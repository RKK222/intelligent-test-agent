## 应用工作区私有 Worktree 与 Diff/推送方案

### Why
- 普通应用工作区 diff 不应调用 opencode /vcs/diff，避免 opencode 服务异常导致"刷新变更列表失败"
- 应用版本进入后应默认切到用户私有 worktree，同一用户同一版本复用 default
- "提交并推送"应把个人 worktree 合并回应用版本分支，冲突在个人 worktree 中解决

### What
后端新增 3 个 Service 方法 + 3 个 API 端点，前端 3 个组件 + 2 个包变更。

#### 后端变更
- **ManagedWorkspaceApplicationService**:
  - `ensureDefaultPersonalWorkspace(versionId, userId, traceId)`: 先查 (versionId, userId, "default")，存在复用，不存在则创建（新命名规则: {branch}_{userId}_default）
  - `getWorkspaceGitDiff(workspaceId, userId)`: 用 GitWorkspaceService.statusPorcelain + diff，不依赖 opencode
  - `publishPersonalWorkspace(personalWorkspaceId, commitMessage, userId, traceId)`: stageAll → commitStaged → push → merge app version branch，冲突返回 CONFLICT + 冲突文件列表

- **ManagedWorkspaceController**: 3 个新端点
  - POST /workspace-versions/{versionId}/ensure-default-personal-workspace
  - GET /workspaces/{workspaceId}/git-diff
  - POST /personal-workspaces/{personalWorkspaceId}/publish

- **ManagedWorkspaceResponses**: 新增 WorkspaceGitDiffResponse、PersonalWorkspacePublishResponse、DefaultPersonalWorkspaceResponse
- **ManagedWorkspaceDtos**: 新增 PublishPersonalWorkspaceRequest

#### 前端变更
- **AgentWorkbench.vue**: handleSelectVersion/pickDefaultWorkspaceForApp/handleCreateVersion 三个入口调用 ensureDefaultPersonalWorkspace
- **GitChangesPanel.vue**: diff 改用 getWorkspaceGitDiff，提交并推送改用 publishPersonalWorkspace
- **FigmaFileExplorer.vue**: 新增 personalWorkspaceId prop 传递
- **backend-api/index.ts**: 新增 3 个 API 方法
- **shared-types/index.ts**: 新增 5 个类型定义

### How
- 私有 worktree 命名: {应用版本分支}_{sanitizedUserId}_default
- 路径: OPENCODE_PERSONAL_WORKTREE_ROOT/{sanitizedVersion}/{sanitizedUserId}/{repoEnglishName}/{sanitizedName}
- 旧 personal workspace 记录不做迁移，新规则仅影响新建的 default/custom 私有空间
- Diff 通过 git status --porcelain 解析变更文件，每个文件调 git diff 获取 patch
- 合并冲突时个人 worktree 进入冲突状态，应用版本副本不受影响
