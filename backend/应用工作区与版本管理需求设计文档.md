# 应用工作区与版本管理需求设计文档

**版本**: v2.2  
**日期**: 2026-06-22  
**前置文档**: `backend/应用隔离与版本管理设计文档.md`（v1.0，存量设计基线）

本文档在 v1.0 的基础上细化工作区目录结构。agent 公共文件提升到应用级 `agents-pub/`，项目代码按分支分化到 `app/`（应用目录）和 `ver/`（版本目录）两类 worktree 中，`private/` 回归应用级统一管理。实现多人并行操作不同版本、不同分支互不干扰。

---

## 1. 差异概览（相对 v1.0）

| 维度 | v1.0（存量设计） | v2.2（本文档） |
|------|-----------------|---------------|
| 目录结构 | 扁平：`{root}/{appId}` | 分层：`{root}/{app-name}/` 下设 `agents-pub/`、`private/`、`app/`、`ver/` |
| agent 组织 | 未定义 | 应用级 `agents-pub/`，每个 agent 一个独立子目录（README + agent.md + skills/ + mcp/） |
| 版本实现 | 同一目录 `git checkout` 切换 | 分支 worktree 分化为 `app/`（主分支）和 `ver/{branch}/`（其他分支），每个独立目录 |
| 多人并行 | 不支持 | `app/` 和 `ver/` 下不同分支 worktree 可被不同用户同时使用 |
| private | 未定义 | 应用级 `private/{userId}/`，所有版本共享用户私有空间 |
| 初始化 | `git clone` + 建目录 | 创建应用目录结构 + 克隆主分支到 `app/` + 部署 agent 到 `agents-pub/` |

## 2. 工作区目录结构

### 2.1 总览

```
/data/workspace/                                    ← 总工作区根路径（可配置）
├── f-gcms/                                         ← 应用目录（以应用名称 slug 为目录名）
│   │
│   ├── agents-pub/                                 ← 公共 agent 目录（应用级，所有版本共享）
│   │   ├── code-reviewer/                          ← agent "code-reviewer" 的独立目录
│   │   │   ├── README.md                           ← 该 agent 的中文说明
│   │   │   ├── agent.md                            ← agent 定义（元信息、系统提示词）
│   │   │   ├── skills/                             ← 该 agent 专有的 skills
│   │   │   │   ├── review-changes.md
│   │   │   │   └── security-check.md
│   │   │   └── mcp/                                ← 该 agent 专有的 MCP 配置（可选）
│   │   │       └── config.json
│   │   ├── test-generator/                         ← agent "test-generator"
│   │   │   ├── README.md
│   │   │   ├── agent.md
│   │   │   └── skills/
│   │   │       ├── unit-test.md
│   │   │       └── integration-test.md
│   │   └── bug-fixer/                              ← agent "bug-fixer"
│   │       ├── README.md
│   │       ├── agent.md
│   │       └── skills/
│   │
│   ├── private/                                    ← 用户私有目录（应用级）
│   │   └── {userId}/                               ← 每个用户的独立私有空间
│   │       ├── settings/                           ← 用户个人设置
│   │       └── workspace/                          ← 用户工作文件（sessions、临时文件等）
│   │
│   ├── app/                                        ← 应用目录（主分支 Git worktree）
│   │   ├── src/                                    ← 项目源码
│   │   ├── pom.xml                                 ← 项目构建文件
│   │   ├── .git                                    ← worktree 的 .git
│   │   └── ...                                     ← 其他项目文件
│   │
│   └── ver/                                        ← 版本目录（其他分支 Git worktree 容器）
│       ├── feature_f-gcms_202606/                  ← 分支 feature/f-gcms_202606 的项目文件
│       │   ├── src/                                ← 该分支的项目源码
│       │   ├── pom.xml
│       │   ├── .git
│       │   └── ...
│       └── hotfix_v1/                              ← 分支 hotfix/v1 的项目文件
│           ├── src/
│           ├── .git
│           └── ...
│
├── another-app/                                    ← 另一个应用
│   ├── agents-pub/
│   ├── private/
│   ├── app/
│   └── ver/
└── ...
```

### 2.2 目录职责说明

| 路径（相对于应用目录） | 职责 | 读写权限 | 生命周期 |
|--------------------------|------|---------|---------|
| `agents-pub/` | 存放所有公共 agent 定义（README、agent.md、skills、mcp） | 管理员可写，普通用户只读 | 初始化时创建，与应用同生命周期 |
| `agents-pub/{agent-name}/README.md` | 该 agent 的中文说明文档 | 管理员可写 | 持续维护 |
| `agents-pub/{agent-name}/agent.md` | agent 定义文件 | 管理员可写 | 持续维护 |
| `agents-pub/{agent-name}/skills/` | 该 agent 专有的 skills | 管理员可写 | 持续维护 |
| `agents-pub/{agent-name}/mcp/` | 该 agent 专有的 MCP 配置 | 管理员可写 | 可选 |
| `private/{userId}/` | 单个用户的私有空间 | 仅该用户可读写 | 用户首次关联应用时创建 |
| `private/{userId}/settings/` | 用户个人设置 | 仅该用户可读写 | 持续更新 |
| `private/{userId}/workspace/` | 用户工作文件 | 仅该用户可读写 | 持续更新 |
| `app/` | 应用主分支（defaultBranch）的 Git worktree，存放项目代码 | 通过 Git 操作变更 | 初始化时创建，应用生命周期 |
| `ver/{branchDir}/` | 其他分支的 Git worktree，每个子目录存该分支的项目代码 | 通过 Git 操作变更 | 新增版本时创建，归档版本时可选清理 |

### 2.3 目录命名规则

| 目录 | 命名规则 | 示例 |
|------|---------|------|
| 应用目录 | 应用名称的 slug 化（小写、连字符与下划线保留） | `f-gcms`、`my_app` |
| agents-pub/ 下子目录 | agent 名称的 slug 化 | `code-reviewer`、`test-generator` |
| ver/ 下子目录 | Git 分支名，`/` 替换为 `_` | `feature/f-gcms_202606` → `feature_f-gcms_202606` |
| private/ 下子目录 | `{userId}`，`usr_` 前缀 | `private/usr_a1b2c3d4` |

### 2.4 设计意图

- **agents-pub 在应用级**：同一应用的所有版本共享相同的 agent 定义，减少冗余；agent 定义与项目代码分离，避免污染 Git 仓库。
- **app/ 与 ver/ 分离**：`app/` 是对外的主分支，"应用目录"代表该项目的主体代码；`ver/` 下的子目录是其他分支的 worktree，"版本目录"承载各类特性/修复分支。两者物理隔离，职责清晰。
- **ver/{branch}/ 直接存项目文件**：`ver/` 下的每个子目录就是一个完整的 Git worktree，打开即用，无需额外嵌套。
- **private 在应用级**：用户设置和私有内容跨版本共享，切换版本不影响用户的个人配置和工作文件。
- **无 private 在版本下**：版本目录只存项目代码，干净纯粹；用户私有数据不随版本扩散。

## 3. 核心实体与关系

### 3.1 实体定义

| 实体 | 业务 ID 前缀 | 核心属性 | 说明 |
|------|-------------|---------|------|
| **Application** | `app_` | name, gitRepoUrl, defaultBranch, workspaceRootPath | 记录应用及其工作区根路径 |
| **ApplicationVersion** | `ver_` | appId, branchName, worktreeType（APP/VER）, worktreePath | 版本 = 分支 = worktree |
| **Workspace** | `wrk_` | appId, versionId | 应用-版本的 workspace 记录 |
| **UserApplication** | — | userId, appId, currentVersionId | 用户-应用-当前版本对照关系 |
| **PublicAgent** | `pag_` | appId, agentName, fileManifest | 应用级 agent 索引（非版本级） |

### 3.2 worktreeType 说明

| worktreeType | 目录位置 | 说明 |
|-------------|---------|------|
| `APP` | `{appRoot}/app/` | 应用主分支 worktree（默认分支），一个应用有且仅有一个 |
| `VER` | `{appRoot}/ver/{branchDir}/` | 版本分支 worktree，可有零至多个 |

### 3.3 路径关系

```
总工作区根路径:      /data/workspace/                          (配置: TEST_AGENT_APPLICATION_WORKSPACE_ROOT)
应用工作区路径:      {root}/{slug(appName)}                     (如 /data/workspace/f-gcms)
agent 公共路径:      {appRoot}/agents-pub/                      (所有版本共享)
用户私有路径:        {appRoot}/private/{userId}/
应用主分支 worktree:  {appRoot}/app/                             (worktreeType=APP)
版本分支 worktree:    {appRoot}/ver/{branchDir}/                 (worktreeType=VER)
agent 工作目录:       即 worktree 所在路径（app/ 或 ver/{branchDir}/）
```

### 3.4 关键约束

- 一个 Application 只有一个 `worktreeType=APP` 的版本（对应 defaultBranch）
- 一个 Application 可以零至多个 `worktreeType=VER` 的版本
- Application 的 workspaceRootPath 初始化后不可变更
- 同一 Application 下分支名唯一（含 worktreeType 维度）
- 同一 `agents-pub/` 下 agent 名称不重复

## 6. 核心业务流程

### 6.1 应用创建

```
输入：name, description, gitRepoUrl, defaultBranch, traceId
前置：操作人具有 APP_ADMIN 或以上角色

流程：
  1. 校验 name 唯一性
  2. 规范校验：gitRepoUrl 格式、分支名字符白名单
  3. 创建 Application 记录（workspace_root_path 暂为空）
  4. 返回 appId

说明：应用创建不触发 workspace 初始化，初始化由独立的 initialize 接口完成。
```

### 6.2 一键初始化应用 workspace

```
输入：appId
前置：操作人具有 APP_ADMIN 或以上角色
      应用状态为 ACTIVE 且 workspace_root_path 为空（未初始化）

流程：
  1. 读取应用信息（gitRepoUrl, defaultBranch, name）
  2. 计算 workspace_root_path = {WORKSPACE_ROOT}/{slug(name)}
     如：/data/workspace/f-gcms
  3. 创建目录骨架：
     {workspace_root_path}/
       agents-pub/
         .gitkeep
       private/
         .gitkeep
       app/                           ← 空目录，待 clone
       ver/                           ← 空容器目录
  4. git clone --branch {defaultBranch} {gitRepoUrl} {workspace_root_path}/app/
     （项目代码克隆到 app/ 目录）
  5. 部署公共 agent 模板到 agents-pub/（见 6.2.1）
  6. 创建 ApplicationVersion 记录（worktree_type=APP, worktree_path=app）
  7. 创建 Workspace 记录（绑定 appId + versionId）
  8. 更新 Application.workspace_root_path
  9. 返回初始化结果

异常：
  Git clone 失败 → INTERNAL_ERROR（脱敏）
  已初始化 → CONFLICT，"该应用已初始化 workspace"
  目录创建失败 → INTERNAL_ERROR
```

#### 6.2.1 公共 agent 部署

初始化时，从【公共 agent 模板库】复制 agent 子目录到 `agents-pub/`。

**模板来源**：配置项 `TEST_AGENT_PUBLIC_AGENT_TEMPLATE_DIR`，默认 `/opt/test-agent/agent-templates/`。

**模板目录结构**（每个 agent 一个独立子目录）：

```
/opt/test-agent/agent-templates/
├── code-reviewer/
│   ├── README.md                    ← 该 agent 的中文说明
│   ├── agent.md                     ← agent 定义
│   ├── skills/
│   │   ├── review-changes.md
│   │   └── security-check.md
│   └── mcp/
│       └── config.json
├── test-generator/
│   ├── README.md
│   ├── agent.md
│   └── skills/
│       ├── unit-test.md
│       └── integration-test.md
└── bug-fixer/
    ├── README.md
    ├── agent.md
    └── skills/
        └── diagnose.md
```

**部署逻辑**：

```
1. 检查模板目录是否存在且非空
2. 遍历模板目录下所有子目录（每个子目录 = 一个 agent）
3. 将整个 agent 子目录复制到 {appRoot}/agents-pub/{agent-name}/
4. 对每个 agent 创建 public_agents 索引记录
5. 若模板目录不存在或为空，仅保留 agents-pub/.gitkeep
```

### 6.3 新增版本

```
输入：appId, branchName, description
前置：操作人具有 APP_ADMIN 或以上角色
      应用已初始化 workspace
      分支名在该应用下不重复
      分支名字符白名单：[a-zA-Z0-9_/-]+

流程：
  1. 校验应用 workspace_root_path 不为空
  2. 计算 branchDirName = 分支名中 / → _
     如：feature/f-gcms_202606 → feature_f-gcms_202606
  3. 在 app/ 目录下执行：
     git worktree add ../ver/{branchDirName} {branchName}
     目标 worktree 路径 = {workspace_root_path}/ver/{branchDirName}/
     （以 app/ 为基础，在新分支上创建 worktree 到 ver/ 下）
  4. 创建 ApplicationVersion 记录：
     worktree_type = VER
     worktree_path = ver/{branchDirName}
  5. 返回 versionId

说明：
  新增版本仅在 ver/ 下创建新 worktree，不重新部署 agent（agent 由 agents-pub/ 全局共享）。
  worktree 目录直接存放项目文件（src/、pom.xml 等），打开即用。

异常：
  分支名重复 → CONFLICT，"该版本已存在"
  Git worktree add 失败 → INTERNAL_ERROR
  应用未初始化 → CONFLICT，"请先初始化应用 workspace"
```

### 6.4 用户关联应用

```
输入：appId
前置：用户已登录

流程：
  1. 校验应用存在且状态为 ACTIVE
  2. 校验用户未重复关联同一应用
  3. 获取应用的默认版本（is_default=true，通常为 worktreeType=APP 的版本）
  4. 创建 UserApplication 记录（current_version_id = 默认版本 ID）
  5. 在 private/{userId}/ 下创建用户私有目录骨架
  6. 返回关联结果
```

### 6.5 用户切换版本

```
输入：appId, versionId
前置：用户已关联该应用

流程：
  1. 校验版本存在且属于该应用
  2. 校验版本状态为 ACTIVE
  3. 更新 UserApplication.currentVersionId → 目标 versionId
  4. 后续 agent 运行时以新版本的 worktree 作为工作目录：
     工作目录 = {Application.workspace_root_path}/{ApplicationVersion.worktree_path}

说明：切换版本仅更新数据库对照关系，即时生效。不同用户可各自切换不同版本互不干扰。

异常：
  版本不存在 → NOT_FOUND
  用户未关联该应用 → FORBIDDEN
  版本已归档 → CONFLICT
```

### 6.6 多人并行场景

```
用户 A 关联 f-gcms，当前版本：app（worktreeType=APP，分支 main）
  → 工作目录：/data/workspace/f-gcms/app/

用户 B 关联 f-gcms，当前版本：ver/feature_f-gcms_202606（worktreeType=VER）
  → 工作目录：/data/workspace/f-gcms/ver/feature_f-gcms_202606/

两者完全独立：
  - 各自在独立的 worktree 目录中发起 agent Run
  - 共享 agents-pub/ 中的 agent 定义
  - 各自的 private/{userId}/ 互不可见
  - 无需切换分支，无文件冲突
```

### 6.7 公共 agent 文件管理

| 操作 | 权限 | 说明 |
|------|------|------|
| 列出应用所有 agent | 所有用户 | 读取 `public_agents` 表，按 `app_id` 过滤 |
| 查看 agent 详情（README、skills 列表） | 所有用户 | 读取 `agents-pub/{agent-name}/` 下文件 |
| 新增 agent（含 README、agent.md、skills） | APP_ADMIN+ | 在 `agents-pub/` 下创建 agent 子目录 + 写索引 |
| 更新 agent 文件 | APP_ADMIN+ | 覆盖 `agents-pub/{agent-name}/` 下文件 + 更新 hash |
| 删除 agent | APP_ADMIN+ | 删除目录 + 标记 public_agents 记录为 ARCHIVED |

## 8. 安全设计

| 关注点 | 措施 |
|--------|------|
| 目录越权 | 所有文件操作校验目标路径在 `{workspace_root_path}` 之内 |
| agents-pub 写保护 | 非 APP_ADMIN 用户不可写 |
| private 目录隔离 | 每个用户只能访问 `private/{selfUserId}/` 下的内容 |
| private 大小限制 | 单用户单应用上限 500MB |
| Git 认证 | 环境变量 `GIT_USERNAME` / `GIT_TOKEN` 注入，不存库 |
| 分支名校验 | 白名单 `[a-zA-Z0-9_/-]+`，上限 255 |
| 日志脱敏 | Git 命令输出不全量记录，错误信息脱敏返回 |
| 审计追踪 | 初始化、版本创建、agent 变更均记录 traceId + 操作人 |
| 越权防护 | Controller 入口校验角色 |
