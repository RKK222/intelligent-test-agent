# 应用工作区与版本管理需求设计文档

**版本**: v2.0  
**日期**: 2026-06-22  
**前置文档**: `backend/应用隔离与版本管理设计文档.md`（v1.0，存量设计基线）

本文档在 v1.0 的基础上细化工作区目录结构，引入 `public/`（公共 agent 文件）和 `private/`（用户私有设置和内容）两级目录隔离，并明确应用 workspace 初始化时公共 agent 的部署流程。

---

## 1. 差异概览（相对 v1.0）

| 维度 | v1.0（存量设计） | v2.0（本文档） |
|------|-----------------|---------------|
| 目录结构 | 扁平：`{root}/{appId}` | 分层：`{root}/{app-name}/` 下设 `public/`、`private/`、`{version-dir}/` |
| 版本实现 | 同一目录内 `git checkout` 切换分支 | 每个版本独立 `git worktree` 子目录 |
| 公共文件 | 未定义 | `public/` 存放 agent 定义、skills、MCP 等公共配置 |
| 用户私有 | 未定义 | `private/{userId}/` 存放用户设置和私有工作内容 |
| 初始化 | 仅 `git clone` + 建目录 | 额外部署公共 agent 文件到 `public/agents/` |

## 2. 工作区目录结构

### 2.1 总览

```
/data/workspace/                           ← 总工作区根路径（可配置）
├── f-gcms/                                ← 应用工作区（以应用名称为目录名）
│   ├── public/                            ← 公共 agent 文件目录
│   │   ├── agents/                        ← agent 定义文件（.md / .json）
│   │   ├── skills/                        ← skill 定义
│   │   ├── mcp/                           ← MCP server 配置
│   │   ├── .opencode.jsonc                ← opencode 公共配置（可选）
│   │   └── README.md                      ← 公共目录说明
│   ├── private/                           ← 用户私有内容目录
│   │   └── {userId}/                      ← 每个用户的独立私有空间
│   │       ├── settings/                  ← 用户个人设置
│   │       └── workspace/                 ← 用户工作文件（sessions 等）
│   ├── main/                              ← 默认版本 worktree（分支 main）
│   │   ├── ...                            ← Git 仓库项目文件
│   │   └── .git                           ← worktree 的 .git 文件（指向主 .git）
│   └── feature_f-gcms_202606/             ← 版本 worktree（分支 feature/f-gcms_202606）
│       ├── ...                            ← 该项目分支的代码
│       └── .git
├── another-app/
│   ├── public/
│   ├── private/
│   └── main/
└── ...
```

### 2.2 目录职责说明

| 路径（相对于应用工作区） | 职责 | 读写权限 | 生命周期 |
|--------------------------|------|---------|---------|
| `public/` | 存放公共 agent 文件、skills、MCP 配置等 | 应用管理员可写，普通用户只读 | 初始化时创建，与应用同生命周期 |
| `public/agents/` | Agent 定义文件 | 同上 | 管理员可通过 API 增删改 |
| `public/skills/` | Skill 定义文件 | 同上 | 同上 |
| `public/mcp/` | MCP server 配置 | 同上 | 同上 |
| `private/` | 用户私有内容根目录 | 系统自动管理 | 初始化时创建 |
| `private/{userId}/` | 单个用户的私有空间 | 该用户可读写 | 用户首次关联应用时创建 |
| `private/{userId}/settings/` | 用户个人设置文件 | 该用户可读写 | 持续更新 |
| `private/{userId}/workspace/` | 用户工作区文件 | 该用户可读写 | 持续更新 |
| `{branch-name}/` | Git worktree 版本目录 | 只读（通过 Git 操作变更） | 新增版本时创建 |

### 2.3 目录命名规则

| 目录 | 命名规则 | 示例 |
|------|---------|------|
| 应用目录 | 应用名称的 slug 化（小写、连字符保留、特殊字符去重） | `f-gcms`、`my-app` |
| 版本目录 | Git 分支名，其中 `/` 替换为 `_` | `main` → `main`；`feature/f-gcms_202606` → `feature_f-gcms_202606` |
| 用户目录 | `private/{userId}`，userId 为 `usr_` 前缀的业务 ID | `private/usr_a1b2c3d4` |

## 3. 核心实体与关系

### 3.1 实体定义（对 v1.0 的补充）

在 v1.0 定义的 Application / ApplicationVersion / UserApplication / Workspace 基础上，新增和明确：

| 实体 | 业务 ID 前缀 | 核心扩展 | 说明 |
|------|-------------|---------|------|
| **Application** | `app_` | 新增 `workspace_root_path` 字段 | 记录应用工作区在磁盘上的根路径，跟随初始化后写入 |
| **ApplicationVersion** | `ver_` | 新增 `worktree_path` 字段 | 记录该版本对应的 git worktree 子目录相对路径 |
| **Workspace** | `wrk_` | 新增 `app_id` / `version_id`（来自 v1.0） | 每个应用有唯一的 workspace 记录 |
| **UserApplication** | — | — | 用户-应用-版本关联 |
| **PublicAgentFile** | `paf_` | 新增实体 | 记录 `public/` 目录下的 agent 文件及其元信息 |

### 3.2 workspace_root_path 与 Workspace 的关系

- **Application.workspace_root_path** = 应用工作区的磁盘根路径，如 `/data/workspace/f-gcms`
- **Workspace.rootPath** = Application.workspace_root_path（两者相等）
- **ApplicationVersion.worktree_path** = 相对于 workspace_root_path 的版本子目录，如 `feature_f-gcms_202606`
- Agent 运行时的工作目录 = `{workspace_root_path}/{worktree_path}` = `/data/workspace/f-gcms/feature_f-gcms_202606`

## 6. 核心业务流程

### 6.1 应用创建

```
输入：name, description, gitRepoUrl, defaultBranch, traceId
前置：操作人具有 APP_ADMIN 或以上角色

流程：
  1. 校验 name 唯一性
  2. 创建 Application 记录（workspace_root_path 暂为空）
  3. 返回 appId

说明：应用创建不触发 workspace 初始化，初始化由独立的 initialize 接口完成。
```

### 6.2 一键初始化应用 workspace

```
输入：appId
前置：操作人具有 APP_ADMIN 或以上角色
      应用状态为 ACTIVE 且 workspace_root_path 为空（未初始化）

流程：
  1. 读取应用信息（gitRepoUrl, defaultBranch, name）
  2. 计算 workspace_root_path = {WORKSPACE_ROOT}/{slug(name)}
     如：/data/workspace/f-gcms
  3. 创建目录结构（见 6.2.1）
  4. 执行 git clone --branch {defaultBranch} {gitRepoUrl} {workspace_root_path}/{branchDirName}
     branchDirName = 分支名中 / → _（如 main → main）
  5. 执行 git worktree 设置（如果默认分支不是裸仓库初始）
  6. 复制公共 agent 模板文件到 public/agents/（见 6.2.2）
  7. 创建 ApplicationVersion 记录（默认版本）
  8. 创建 Workspace 记录（绑定 appId + versionId）
  9. 更新 Application.workspace_root_path
  10. 返回初始化结果

异常：
  Git clone 失败 → INTERNAL_ERROR
  已初始化 → CONFLICT，"该应用已初始化 workspace"
  目录创建失败 → INTERNAL_ERROR
```

#### 6.2.1 目录初始化明细

```
{workspace_root_path}/
  public/
    agents/
      .gitkeep                  ← 占位文件，agent 部署后替换
    skills/
      .gitkeep
    mcp/
      .gitkeep
    README.md                   ← 模板说明文件
  private/
    .gitkeep
```

#### 6.2.2 公共 agent 部署

初始化时，系统从【公共 agent 模板库】中复制默认 agent 定义文件到 `public/agents/` 目录。

**模板来源**：配置项 `TEST_AGENT_PUBLIC_AGENT_TEMPLATE_DIR`，指向服务器上的公共 agent 模板目录，默认路径为 `/opt/test-agent/agent-templates/`。

**部署逻辑**：
```
1. 检查模板目录是否存在且非空
2. 将模板目录下所有 .md / .json / .yaml 文件复制到 public/agents/
3. 对每个文件创建 PublicAgentFile 索引记录
4. 如果模板目录不存在或为空，仅在 public/agents/ 下创建 .gitkeep 占位
```

**示例模板目录结构**：
```
/opt/test-agent/agent-templates/
  ├── code-reviewer.md          ← 代码审查 agent
  ├── test-generator.md          ← 测试用例生成 agent
  ├── bug-fixer.md              ← 缺陷修复 agent
  └── doc-writer.md             ← 文档编写 agent
```

#### 6.2.3 初始化成功响应

```json
{
  "appId": "app_xxx",
  "appName": "f-gcms",
  "workspaceId": "wrk_xxx",
  "versionId": "ver_xxx",
  "workspaceRootPath": "/data/workspace/f-gcms",
  "versionPath": "/data/workspace/f-gcms/main",
  "publicAgents": [
    { "fileName": "code-reviewer.md", "filePath": "agents/code-reviewer.md" },
    { "fileName": "test-generator.md", "filePath": "agents/test-generator.md" }
  ]
}
```

### 6.3 新增版本

```
输入：appId, branchName, description
前置：应用已初始化 workspace
      分支名在该应用下不重复
      分支名字符白名单：[a-zA-Z0-9_/-]+

流程（Git worktree 方式）：
  1. 校验应用 workspace_root_path 不为空
  2. 在应用工作区下执行：
     git worktree add {branchDirName} {branchName}
     （以 main worktree 为基础，checkout 新分支）
  3. 创建 ApplicationVersion 记录（worktree_path = branchDirName）
  4. 返回 versionId

说明：每个版本以独立 worktree 目录存在，agent 运行时工作目录 = {workspace_root_path}/{worktree_path}。

异常：
  分支名重复 → CONFLICT
  Git 操作失败 → INTERNAL_ERROR
  应用未初始化 → CONFLICT，"请先初始化应用 workspace"
```

### 6.4 用户关联应用

```
输入：appId
前置：用户已登录

流程：
  1. 校验应用存在且状态为 ACTIVE
  2. 校验用户未重复关联同一应用
  3. 获取应用的默认版本
  4. 创建 UserApplication 记录（current_version_id = 默认版本）
  5. 在 private/{userId}/ 下创建用户私有目录
  6. 返回关联结果
```

### 6.5 用户切换版本

```
输入：appId, versionId
前置：用户已关联该应用

流程：
  1. 校验版本存在且属于该应用
  2. 更新 UserApplication.currentVersionId
  3. 后续 agent 运行时自动使用新版本的 worktree_path 作为工作目录

说明：切换版本仅更新数据库关联记录，不涉及 git checkout，因为版本已以 worktree 形式存在。

异常：
  版本不存在 → NOT_FOUND
  用户未关联该应用 → FORBIDDEN
```

### 6.6 公共 agent 文件管理

| 操作 | 权限 | 说明 |
|------|------|------|
| 列出席应用的所有公共 agent 文件 | 所有用户 | 读取 `public_agent_files` 表 |
| 上传/新增公共 agent 文件 | APP_ADMIN+ | 写入 `public/agents/` 磁盘 + 创建索引记录 |
| 删除公共 agent 文件 | APP_ADMIN+ | 删除磁盘文件 + 标记索引记录为 ARCHIVED |
| 读取 agent 文件内容 | 所有用户 | 读取 `public/` 目录下文件 |

## 8. 安全设计

| 关注点 | 措施 |
|--------|------|
| 目录越权 | 所有文件操作必须校验目标路径在 `{workspace_root_path}` 之内 |
| public 目录写保护 | 非 APP_ADMIN 用户对 `public/` 不可写 |
| private 目录隔离 | 每个用户只能访问 `private/{selfUserId}/` 下的内容 |
| Git 认证凭据 | 通过环境变量 `GIT_USERNAME` / `GIT_TOKEN` 注入，不存数据库 |
| 分支名校验 | 白名单 `[a-zA-Z0-9_/-]+`，拒绝命令注入 |
| 日志脱敏 | Git 命令输出不全量记录，错误信息脱敏后返回 |
| 审计追踪 | 初始化、版本创建、agent 文件变更须记录 traceId + 操作人 |
| 越权防护 | Controller 入口校验角色，不依赖前端 |
