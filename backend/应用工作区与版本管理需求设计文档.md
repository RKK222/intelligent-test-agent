# 应用工作区与版本管理需求设计文档

**版本**: v2.3  
**日期**: 2026-06-22  
**前置文档**: `backend/应用隔离与版本管理设计文档.md`（v1.0，存量设计基线）

核心设计原则：agent 公共文件提升到总工作区根路径全局共享；应用目录下仅含版本目录，版本目录直接承载项目文件。

---

## 1. 差异概览（相对 v1.0）

| 维度 | v1.0（存量设计） | v2.3（本文档） |
|------|-----------------|---------------|
| 目录结构 | `{root}/{appId}` | `{root}/agents-pub/`（全局）+ `{root}/{app}/{version}/`（项目文件） |
| agent 组织 | 未定义 | 总工作区级 `agents-pub/`，每个 agent 独立子目录（README + agent.md + skills + mcp） |
| 版本实现 | 同一目录 `git checkout` | 应用下仅含版本目录（如 `202606`），每版本 = 独立 Git worktree |
| 多人并行 | 不支持 | 不同用户可绑定不同版本，并行工作互不干扰 |
| 用户私有 | 未定义 | 不设用户私有目录 |

## 2. 工作区目录结构

### 2.1 总览

```
/data/workspace/                                    ← 总工作区根路径（可配置）
│
├── agents-pub/                                     ← 公共 agent 目录（总工作区级，所有应用共享）
│   ├── code-reviewer/                              ← agent "code-reviewer"
│   │   ├── README.md                               ← 该 agent 的中文说明
│   │   ├── agent.md                                ← agent 定义（元信息、系统提示词）
│   │   ├── skills/                                 ← 该 agent 专有的 skills
│   │   │   ├── review-changes.md
│   │   │   └── security-check.md
│   │   └── mcp/                                    ← 该 agent 专有的 MCP 配置（可选）
│   │       └── config.json
│   ├── test-generator/                             ← agent "test-generator"
│   │   ├── README.md
│   │   ├── agent.md
│   │   └── skills/
│   │       ├── unit-test.md
│   │       └── integration-test.md
│   └── bug-fixer/                                  ← agent "bug-fixer"
│       ├── README.md
│       ├── agent.md
│       └── skills/
│
├── f-gcms/                                         ← 应用目录（应用名称 slug）
│   ├── 202606/                                     ← 版本目录
│   │   ├── src/                                    ← 应用项目文件
│   │   ├── pom.xml
│   │   ├── .git
│   │   └── ...                                     ← 其他项目文件
│   └── 202607/                                     ← 版本目录
│       ├── src/
│       ├── pom.xml
│       ├── .git
│       └── ...
│
└── another-app/
    ├── 202606/
    └── 202608/
```

### 2.2 目录职责说明

| 路径 | 职责 | 生命周期 |
|------|------|---------|
| `/data/workspace/agents-pub/` | 公共 agent 全局定义，所有应用共享 | 随平台部署创建，持续维护 |
| `/data/workspace/agents-pub/{agent-name}/README.md` | 该 agent 的中文说明 | 持续维护 |
| `/data/workspace/agents-pub/{agent-name}/agent.md` | agent 定义文件 | 持续维护 |
| `/data/workspace/agents-pub/{agent-name}/skills/` | 该 agent 专有的 skills | 持续维护 |
| `/data/workspace/agents-pub/{agent-name}/mcp/` | 该 agent 专有的 MCP 配置 | 可选 |
| `/data/workspace/{app}/` | 应用目录，仅作版本容器 | 应用创建 → 归档 |
| `/data/workspace/{app}/{version}/` | 版本目录，即为该版本的项目 worktree | 初始化/新增版本时创建 |

### 2.3 命名规则

| 目录 | 规则 | 示例 |
|------|------|------|
| 应用目录 | 应用名称 slug 化 | `f-gcms`、`my-app` |
| 版本目录 | 版本标识（由管理员定义，字符串） | `202606`、`v2.0`、`feature-xxx` |
| agent 子目录 | agent 名称 slug 化 | `code-reviewer` |

## 3. 核心实体与关系

### 3.1 实体定义

| 实体 | 业务 ID 前缀 | 核心属性 | 说明 |
|------|-------------|---------|------|
| **Application** | `app_` | name, gitRepoUrl, workspaceRootPath | 应用及其工作区根路径 |
| **ApplicationVersion** | `ver_` | appId, versionName, branchName, worktreePath | 版本 = worktree 目录 |
| **UserApplication** | — | userId, appId, currentVersionId | 用户-应用-当前版本关联 |
| **Workspace** | `wrk_` | appId, versionId | 应用-版本 workspace 记录 |
| **PublicAgent** | `pag_` | agentName, fileManifest | 全局 agent 索引（总工作区级） |

### 3.2 路径关系

```
总工作区根:         /data/workspace/                    (TEST_AGENT_APPLICATION_WORKSPACE_ROOT)
全局 agent:         {root}/agents-pub/                   (所有应用共享)
应用目录:           {root}/{slug(appName)}/              (/data/workspace/f-gcms/)
版本目录:           {appDir}/{versionName}/              (/data/workspace/f-gcms/202606/)
agent 工作目录:      = 版本目录（agent run 时的工作目录）
```

## 4. 数据库设计

### 4.1 新增表：applications（应用表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增 surrogate PK |
| app_id | VARCHAR(128) | 业务 ID，前缀 `app_`，唯一 |
| name | VARCHAR(255) | 应用名称，不可空，唯一 |
| description | TEXT | 应用描述，可空 |
| git_repo_url | TEXT | Git 仓库地址，不可空 |
| workspace_root_path | VARCHAR(1024) | 应用目录路径（如 `/data/workspace/f-gcms`），初始化后写入 |
| status | VARCHAR(32) | ACTIVE / ARCHIVED |
| created_by | VARCHAR(128) | 创建人用户 ID |
| trace_id | VARCHAR(128) | 操作追踪 ID |
| created_at | TIMESTAMP | — |
| updated_at | TIMESTAMP | — |

### 4.2 新增表：application_versions（应用版本表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增 surrogate PK |
| version_id | VARCHAR(128) | 业务 ID，前缀 `ver_`，唯一 |
| app_id | VARCHAR(128) | 所属应用 ID |
| version_name | VARCHAR(128) | 版本标识（即版本目录名，如 `202606`），不可空 |
| branch_name | VARCHAR(255) | 关联的 Git 分支名，可空（初始化可不指定分支） |
| worktree_path | VARCHAR(1024) | worktree 完整相对路径（如 `202606`） |
| description | TEXT | 版本描述，可空 |
| status | VARCHAR(32) | ACTIVE / ARCHIVED |
| created_by | VARCHAR(128) | 创建人用户 ID |
| trace_id | VARCHAR(128) | 操作追踪 ID |
| created_at | TIMESTAMP | — |
| updated_at | TIMESTAMP | — |

唯一约束：`(app_id, version_name)`。

### 4.3 新增表：user_applications（用户-应用关联表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增 surrogate PK |
| user_id | VARCHAR(128) | 用户 ID |
| app_id | VARCHAR(128) | 应用 ID |
| current_version_id | VARCHAR(128) | 当前选中版本 ID |
| created_at | TIMESTAMP | — |
| updated_at | TIMESTAMP | — |

唯一约束：`(user_id, app_id)`。

### 4.4 新增表：public_agents（公共 agent 索引表）

agent 文件存储在 `/data/workspace/agents-pub/` 下，不与特定应用绑定。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 自增 surrogate PK |
| agent_id | VARCHAR(128) | 业务 ID，前缀 `pag_`，唯一 |
| agent_name | VARCHAR(255) | agent 名称（agents-pub 下的子目录名），唯一 |
| directory_path | VARCHAR(1024) | agent 目录相对路径（如 `code-reviewer`） |
| file_manifest | TEXT | agent 目录内文件清单（JSON） |
| created_by | VARCHAR(128) | 创建人用户 ID |
| trace_id | VARCHAR(128) | 操作追踪 ID |
| created_at | TIMESTAMP | — |
| updated_at | TIMESTAMP | — |

### 4.5 现有表扩展

**workspaces 表新增字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| app_id | VARCHAR(128) | 所属应用 ID，可空 |

## 6. 核心业务流程

### 6.1 应用创建

```
输入：name, description, gitRepoUrl, traceId
前置：操作人具有 APP_ADMIN 或以上角色

流程：
  1. 校验 name 唯一性
  2. 校验 gitRepoUrl 格式
  3. 创建 Application 记录（workspace_root_path 暂为空）
  4. 返回 appId
```

### 6.2 一键初始化应用 workspace

```
输入：appId, versionName（如 "202606"）
前置：操作人具有 APP_ADMIN 或以上角色
      应用状态为 ACTIVE 且 workspace_root_path 为空（未初始化）

流程：
  1. 读取应用信息（gitRepoUrl, name）
  2. 计算 workspace_root_path = {WORKSPACE_ROOT}/{slug(name)}
     如：/data/workspace/f-gcms
  3. 创建目录：{workspace_root_path}/{versionName}/
     如：/data/workspace/f-gcms/202606/
  4. git clone {gitRepoUrl} {workspace_root_path}/{versionName}/
     （项目文件直接克隆到版本目录下）
  5. 创建 ApplicationVersion 记录（version_name = versionName, worktree_path = versionName）
  6. 创建 Workspace 记录（绑定 appId + versionId）
  7. 更新 Application.workspace_root_path
  8. 返回初始化结果

异常：
  Git clone 失败 → INTERNAL_ERROR（脱敏）
  已初始化 → CONFLICT
  目录创建失败 → INTERNAL_ERROR
```

#### 6.2.1 初始化成功响应

```json
{
  "appId": "app_xxx",
  "appName": "f-gcms",
  "workspaceId": "wrk_xxx",
  "versionId": "ver_xxx",
  "versionName": "202606",
  "workspaceRootPath": "/data/workspace/f-gcms",
  "versionPath": "/data/workspace/f-gcms/202606"
}
```

### 6.3 新增版本

```
输入：appId, versionName（如 "202607"）, branchName（可选）, description
前置：操作人具有 APP_ADMIN 或以上角色
      应用已初始化 workspace
      versionName 在该应用下不重复

流程：
  1. 校验应用 workspace_root_path 不为空
  2. 在工作区下执行：
     cd {workspace_root_path}/{已有版本目录}
     git worktree add ../{versionName} {branchName 或 新建分支}
     目标路径 = {workspace_root_path}/{versionName}/
  3. 创建 ApplicationVersion 记录
  4. 返回 versionId

异常：
  versionName 重复 → CONFLICT
  Git worktree add 失败 → INTERNAL_ERROR
  应用未初始化 → CONFLICT
```

### 6.4 用户关联应用

```
输入：appId
前置：用户已登录

流程：
  1. 校验应用存在且状态为 ACTIVE
  2. 校验用户未重复关联同一应用
  3. 获取应用的第一个可用版本作为默认版本
  4. 创建 UserApplication 记录（current_version_id = 默认版本 ID）
  5. 返回关联结果
```

### 6.5 用户切换版本

```
输入：appId, versionId
前置：用户已关联该应用

流程：
  1. 校验版本存在且属于该应用、状态为 ACTIVE
  2. 更新 UserApplication.currentVersionId
  3. agent 下次运行时工作目录 = {workspace_root_path}/{version.worktree_path}

说明：切换仅更新数据库记录，即时生效。
```

### 6.6 公共 agent 管理

agent 全局存储在 `/data/workspace/agents-pub/` 下，不与特定应用绑定。

| 操作 | 权限 | 说明 |
|------|------|------|
| 列出所有公共 agent | 所有用户 | 读取 `public_agents` 表 |
| 查看 agent 详情 | 所有用户 | 读取 `agents-pub/{agent-name}/` 下文件 |
| 新增 agent | SUPER_ADMIN / SYSTEM_ADMIN | 在 `agents-pub/` 下创建子目录 + 写索引 |
| 更新 agent | SUPER_ADMIN / SYSTEM_ADMIN | 更新文件 + 索引 |
| 删除 agent | SUPER_ADMIN / SYSTEM_ADMIN | 删除目录 + 标记 ARCHIVED |

## 8. 安全设计

| 关注点 | 措施 |
|--------|------|
| 目录越权 | 文件操作必须校验目标路径在 `{workspace_root_path}` 内 |
| agents-pub 写保护 | 仅 SUPER_ADMIN / SYSTEM_ADMIN 可写 |
| Git 认证 | 环境变量注入，不存库 |
| 版本名校验 | 字母数字短横线下划线，上限 128 |
| 日志脱敏 | Git 输出不全量记录，错误信息脱敏 |
| 审计追踪 | 初始化、版本创建、agent 变更记录 traceId + 操作人 |
| 越权防护 | Controller 入口校验角色 |
