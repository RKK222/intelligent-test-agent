# Phase 10 端到端加固和发布验收

## 阶段目标

完成从本地开发到可发布版本的质量加固，包括安全、性能、兼容性、可观测性、端到端测试、构建产物和部署文档。

## 可验收功能清单

1. 后端唯一可部署包 `test-agent-app` 可构建和启动。
2. 前端 `agent-web` 可构建生产产物。
3. 端到端测试覆盖核心闭环。
4. 安全、限流、日志脱敏和密钥配置通过检查。
5. API、事件、数据库字段兼容策略明确。
6. 前后端性能保护通过基准场景验证。

## 修改项目

- `backend/*`
- `frontend/*`
- `tools/*`
- `docs/*`
- 部署配置和发布说明

## 实现功能

- 后端构建、测试、镜像或部署包脚本稳定。
- 前端构建、静态资源、环境变量和部署脚本稳定。
- 端到端场景覆盖创建工作区、打开文件、发起任务、查看事件、接受 Diff、运行测试、查看报告。
- 关键链路都有 traceId、日志和指标。
- 大文件、大 Diff、高频 RunEvent、长日志有性能保护。

## 验收方式

- 后端 `mvn clean package` 通过。
- 前端 lint、typecheck、unit test、e2e test 和 production build 通过。
- 安全文档和配置检查通过。
- 性能验收记录首屏、SSE 延迟、大文件和大 Diff 场景。
- 发布说明列出版本、配置、已知限制和回滚方式。
