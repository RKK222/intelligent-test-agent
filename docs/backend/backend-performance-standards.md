# 后端性能规范

本规范约束后端 API、数据库、opencode 调用和事件流的性能边界。

## API

1. 列表接口必须分页或设置明确上限。
2. 查询参数必须限制最大 page size。
3. 大对象下载、日志、事件回放不得一次性全部加载到内存。
4. 同步 API 不执行不可控耗时任务，长任务应异步化或流式返回。
5. Workspace 文件目录列表必须单层读取并设置上限，默认不超过 1000 项。
6. Workspace 文件内容读写必须限制大小，默认不超过 1MB，避免把大文件一次性加载到 API 响应。

## 数据库

1. 避免 N+1 查询。
2. 高频查询必须有索引设计。
3. 批量写入使用批处理或明确事务边界。
4. 查询必须只取需要字段，避免无意义加载大文本或大 JSON。
5. JDBC 连接池统一使用 Druid，连接池大小通过 `TEST_AGENT_DB_POOL_INITIAL_SIZE`、`TEST_AGENT_DB_POOL_MIN_IDLE`、`TEST_AGENT_DB_POOL_MAX_ACTIVE`、`TEST_AGENT_DB_POOL_MAX_WAIT_MILLIS` 配置；不得在代码中硬编码环境容量。

## opencode 调用

1. 外部调用必须有连接、读取和整体超时。
2. 重试必须有上限，只对可重试错误执行。
3. 取消 Run、SSE 断开、请求超时必须释放资源。
4. opencode server 节点选择不得每次全量扫描大表。

## SSE 与事件

1. SSE 输出必须考虑背压和客户端断开。
2. 事件回放必须基于 runId 和 seq 增量查询。
3. 高频事件可以合并或节流，但不能丢失必要状态。
4. Last-Event-ID 续传必须避免重复发送不可幂等事件。
5. Phase 04 SSE polling 默认批量读取 100 条事件，轮询间隔默认 500ms；调整这些值必须同时评估数据库读压和前端渲染频率。

## 日志与指标

1. 热路径日志默认只记录关键字段，不记录大请求体或大响应体。
2. 指标标签不得使用高基数字段，例如完整 message、token、用户输入。
3. 性能问题排查优先使用 traceId、结构化日志和 Micrometer 指标。
