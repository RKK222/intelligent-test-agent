# 前端性能规范

本规范适用于完全自研的 `frontend/` 工程。

## 首屏与加载

1. 首屏只加载工作台启动必需数据。
2. 大列表、大文件树、长日志和长报告必须分页、虚拟化或懒加载。
3. 非首屏能力按需加载，例如报告详情、Skill Studio、截图预览。
4. 避免在启动阶段并发请求无关接口。
5. Next.js 路由和 package 拆分必须控制首屏 bundle。

## RunEvent SSE 渲染

1. 高频事件不能逐条触发重型渲染。
2. 事件处理应批量合并、节流或使用局部状态更新。
3. 断开 SSE 时必须释放订阅、定时器和缓存引用。
4. `Last-Event-ID` 恢复不能导致重复渲染不可幂等 UI。
5. 日志类输出必须限制 DOM 节点数量，必要时使用虚拟列表。
6. 当前事件流应优先按 `eventId` 去重，兼容旧事件时才回退 `runId + seq`；Agent 卡片和日志只追加必要展示字段，`seq=0` transient 文本事件不能因为相同 seq 被错误丢弃。

## 工作台与编辑器

1. Dockview 面板恢复不能阻塞首屏交互。
2. Monaco 编辑器和 Diff 组件必须按需加载。
3. 大文件打开前应有大小检查和只读策略。
4. Diff 展示必须避免一次性渲染超大变更。
5. 面板切换时不得重复初始化重型实例。
6. Phase 06-08 中 Monaco 编辑器和 Monaco Diff 必须维持懒加载，不能进入首屏同步 bundle。

## 请求与缓存

1. 相同资源避免重复请求。
2. 请求必须支持取消或过期保护。
3. 页面切换时释放无用请求和订阅。
4. 缓存必须有失效策略，不能展示明显过期的 Run 状态。
5. TanStack Query key 必须稳定，避免缓存错乱。

## Bundle

1. 功能包按 Web IDE 能力拆分。
2. 大型依赖必须评估包体影响。
3. `ui-kit` 不引入与现有设计系统冲突的重量级依赖。
4. `shared-types` 只放类型和轻量常量，不引入运行时大依赖。

## 可观测性

前端关键流程应能记录：

- API 请求耗时和失败。
- RunEvent SSE 连接状态。
- 事件处理延迟。
- 页面核心交互错误。
- 大文件、大 Diff、长日志的性能保护触发情况。
