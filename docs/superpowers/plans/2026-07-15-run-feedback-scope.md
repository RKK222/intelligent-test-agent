# 按 Run 提供整体评价实施计划

## 实施项

1. 新增 Run 反馈领域对象、Repository 与应用服务，覆盖成功状态、主对话、归属、幂等和 100 条批量限制。
2. 新增 Run 反馈 HTTP 写入、单查与批查 API；旧消息 API 在可关联时委托 Run 服务。
3. 通过 Flyway 回填、去重并演进反馈约束；新增 MyBatis Run 反馈单查/批查和 Run 状态批查。
4. 前端消息、reducer 和时间线投影按用户轮保存 `runId/runStatus`，完成状态插槽透出当前工作状态行。
5. 工作台按 Run 批量加载、提交和回显；完成入口不依赖 assistant 消息，无输出和多轮历史均可评价。
6. 同步 API、事件、数据库、模块 README 与会话日志，执行后端/前端定向和完整验证。

## 验收

- 成功主 Run 在最后输出下方展示状态图标与反馈；无 assistant part 仍展示。
- 新一轮开始后所有历史成功 Run 继续保留独立入口与选中/提交状态。
- 非成功、状态未知、旁路问答及子 Agent 不展示；旧消息接口继续兼容。
- 新反馈表事实满足单用户单 Run 唯一，批量历史加载无 N+1。
