# 用户消息浅灰背景与视觉间距设计

## 背景与目标

主对话时间线中的用户消息气泡目前使用白色背景，用户消息行下方仅保留 6px 间距。根据浏览器批注，需要用浅灰色增强用户消息的来源辨识度，并在用户消息与后续智能体过程或回答之间保留少量空白。

## 范围

- 只修改 `frontend/packages/agent-chat` 的 opencode-like 主时间线用户消息样式。
- 用户消息继续右对齐，气泡宽度、文字、复制按钮、圆角、边框和工作区上下文 chip 行为不变。
- 智能体消息、时间线投影、折叠状态、运行状态和消息顺序不变。
- 不修改 `FigmaChatPanel.vue` 中已作废的旧消息循环。
- 不修改 API、RunEvent、DTO、数据库、后端、安全策略或环境配置。

## 视觉设计

用户气泡背景从白色 `#ffffff` 改为现有语义 token `var(--oc-surface-muted)`。该 token 当前默认值为 `#f2f4f7`，并可继续由外层 `--ta-chat-message-bg` 主题变量覆盖。保留现有细边框，避免浅灰气泡与页面背景之间失去边界。

用户消息行的下边距从 6px 调整为 `var(--oc-space-3)`，当前值为 12px。上边距和左右布局保持不变，因此新增空白只出现在用户消息下方，用于与紧随其后的智能体消息分隔。

不增加伪元素、额外 DOM 或新的颜色变量，避免为单一视觉调整引入额外结构和主题分支。

## 测试与验收

- 先增加样式契约测试，验证用户气泡使用 `var(--oc-surface-muted)`，用户消息行使用 `margin: 4px 0 var(--oc-space-3)`；实现前运行并确认测试因现有白色背景和 6px 下边距而失败。
- 完成最小 CSS 修改后运行 agent-chat 定向 Vitest、前端 lint、typecheck、全量 Vitest 和生产 build。
- 页面验收确认用户气泡呈浅灰色，气泡下方与智能体内容之间存在小幅空白，且用户消息仍保持右对齐。
