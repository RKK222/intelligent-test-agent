# 可拖动彩色小宠物与进程状态锚定展开设计

## 背景与目标

工作台顶部的 MIMO 小宠物当前为深色 SVG，会在空闲时随机移动和跳跃，但用户不能调整它的位置。右侧聊天面板的 TestAgent 进程状态已可收起为一个可拖动的绿色小圆点；不过展开状态卡仍处于聊天面板的普通文档流中，因此不会跟随小圆点的位置。

本次目标是在不改变现有进程查询、初始化或聊天业务逻辑的前提下：

1. 将小宠物改为低饱和双色点缀的彩色形象，并让用户可拖动定位；
2. 用户拖动过小宠物后，停止它的随机行动，避免位置被后续动画覆盖；
3. 收起后的绿色进程圆点在哪里，展开后的进程状态卡就锚定在哪里，并在视口边缘自动换向以完整显示；
4. 两类位置都仅保存为浏览器本地偏好，不写入后端。

## 方案选择

采用“深蓝灰主体 + 青绿、紫色点缀”的小宠物方案。主体继续与工作台中性视觉协调，眼睛、天线端点和少量机身高光使用低饱和渐变；不采用整机彩虹或多分区高饱和配色。

## 交互与状态

### 小宠物

- 宠物出现后保持原有随机行为，直到用户完成一次有效拖动。
- 使用 Pointer Events：鼠标、触控和触控笔均可拖动；小幅位移按点击处理，不暂停行为。
- 有效拖动后，将左上角视口坐标夹在宠物尺寸和安全边距内，写入 `localStorage`；刷新时以该位置显示宠物。
- 用户定位后清理宠物行为定时器、停止后续随机动作，当前次拖动结束时进入静止状态。窗口缩放时继续把已保存位置夹在可视区域内。
- 宠物根元素恢复 pointer events，仅在拖动过程中禁用页面文字选择并使用 `grabbing` 光标；无效点击不影响页面其他操作。
- 顶部栏右侧提供 `data-testid="robot-visibility-toggle"` 的宠物开关，展开时 `aria-label="收起小宠物"`、收起时 `aria-label="唤起小宠物"`，并用 `aria-pressed` 表示可见状态；点击不关闭现有头部菜单。
- 开关可立即唤起静止宠物或将其收起。自动规则保持为页面连续 60 秒无鼠标、键盘、滚动或点击且窗口聚焦时自行跳出；手动收起后重新开始这段空闲计时。若宠物已有手动保存位置，自动重新出现时静止地恢复在该位置，继续遵守“手动定位后不随机移动”。

### 进程状态

- 收起绿点继续兼容既有 `figma-chat-process-dot-pos` 本地偏好；新写入使用版本化格式 `{ v: 2, cardX, cardY, dotSide }`：卡片锚点是唯一坐标，`dotSide` 只表示绿点相对卡片的四个方位而不保存第二组坐标。旧 `{x,y}` 绿点坐标在读取时转换为卡片锚点和方位、再重写为 v2，保留边缘用户看到的绿点位置。
- 已验证的 v2 锚点在卡片尚未真实测量前不得用 fallback 尺寸夹取或覆盖；首次实际展开并测量后才按真实尺寸夹取和必要时持久化。绿点视觉固定为 8px 半透明柔光点与无实心中心的淡色光晕，避免实心按钮感。
- 进程状态有明确的“默认内嵌”和“用户浮动”模式：没有有效的已保存拖动坐标时，状态卡默认显示在聊天区域顶部的原初始化位置。用户在本次会话中收起它时，才临时改为浮点；只有有效拖动才持久化坐标，刷新后没有保存坐标仍回到默认内嵌模式。
- 用户浮动模式以卡片左上角锚点为唯一坐标事实源。展开卡从内联状态拖动越过阈值时，直接把 `getBoundingClientRect()` 的 left/top 写为锚点，首帧不跳动；绿点位置由锚点、卡片尺寸、间距和视口安全边距派生。
- 绿点与展开卡拖动都只移动同一张卡片锚点；窗口缩放、卡片尺寸变化和后续拖动统一夹取这张锚点，再重新派生绿点，因此不会出现卡片与绿点各自夹取不同步。拖动后的点击抑制只覆盖同一次浏览器合成 click，pointerup/cancel 后必须自动释放，不能吞掉下一次真实点击。点击展开卡的非拖动操作（初始化按钮除外）仍收起，Enter/Space 语义不变。

## 兼容性与非功能要求

- 仅使用 Chromium 108 已支持的 Pointer Events、`getBoundingClientRect`、CSS 固定定位和 `localStorage`。
- 浏览器禁用或拒绝 localStorage 时退回默认位置且不抛错。
- 进程状态卡的既有 ready/non-ready、初始化按钮、键盘操作和无障碍名称必须保持。
- 不记录用户位置到日志，也不发送网络请求。

## 验收标准

1. 小宠物可见时呈现低饱和双色点缀而非纯黑；顶部开关可立即唤起/收起，连续 60 秒空闲时也会自动跳出；拖动后停在释放位置，刷新后仍使用该位置且不再随机移动。
2. 初始状态卡位于聊天区域顶部；绿点为 8px 半透明柔光点。绿点和展开卡均可拖动，用户移动后展开卡贴近该点而非回到固定位置；未拖动时刷新仍回到顶部默认卡片。
3. 子 Agent 视图中不显示绿点或状态卡；返回主 Agent 后按原模式恢复。
3. 把绿点放在右下、右上、左下等边缘时，展开卡自动向有空间的一侧显示且不超出视口。
4. 定向 Vitest、前端 lint/typecheck/build 通过；实际启动本地前端后可手工验证两种拖动与展开行为。
