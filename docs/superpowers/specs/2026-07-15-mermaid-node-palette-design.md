# Mermaid 节点图形库交互设计

## 目标与范围

调整 Flowchart/Graph 可视化编辑器的节点创建与属性区域，使用户可以先识别节点外形，再把所需节点类型直接拖入画布。右侧属性栏改为“固定节点图形库 + 当前节点属性”结构；顶部删除“新增节点”和“新增连线”，同时删除整块连线列表。Sequence Diagram 编辑器不在本次范围内。

Markdown 仍是唯一事实源。本次不修改 Mermaid 领域模型、parser、serializer、保存链路、后端 API、事件、数据库或全局状态。

## 交互设计

顶部工具栏只保留“自动布局”和“图方向”。右侧上半部固定展示矩形、圆角、胶囊、判断、圆形五种节点卡片，卡片轮廓与画布节点形状一致，并同时显示中文名称。

用户把卡片拖到画布时，编辑器将浏览器指针坐标换算为 Vue Flow 图坐标，在落点创建对应类型的“新节点”，并分配当前图内唯一的 `N<number>` ID。新节点创建后自动进入选中状态。为了让键盘用户也能完成同一操作，卡片同时作为可聚焦按钮；点击或按键激活时，在画布当前可见区域的默认位置创建节点。

右侧图形库下方是上下文属性区：未选中节点时显示操作提示；选中节点时显示只读节点 ID、可编辑节点名称、节点类型选择和删除按钮。修改和删除沿用当前草稿更新逻辑，删除节点时继续同步删除关联边。

连线不再提供顶部快捷创建或右侧批量编辑。用户只通过节点 Handle 拖出连线；已有连线继续在画布展示，并由现有领域模型和 serializer 原样处理。

## 实现边界

改动集中在 `frontend/packages/editor/src/mermaid/visual-editor/MermaidVisualEditor.vue` 及其组件测试。Vue Flow 实例提供浏览器坐标到图坐标的转换，拖放逻辑只负责创建领域节点，不直接操作 Vue Flow 内部节点数组。节点 ID 分配和草稿克隆保持纯前端、确定性且不修改传入 prop。

视觉上继续复用 editor 的 `--ta-*` token。图形库采用紧凑的两列卡片布局，以节点真实轮廓作为唯一强调元素；拖拽中、悬停、键盘焦点和画布可放置状态提供清晰反馈。窄屏下右栏沿用现有下置布局并可滚动，不引入新的全局样式或依赖。

## 测试与验收

组件测试覆盖：顶部两个新增按钮不存在；右栏展示五种可识别节点类型；拖放按换算后的画布坐标创建正确类型节点；点击卡片可创建节点；新节点自动选中；选中既有节点后可修改名称、类型并删除关联边；Vue Flow Handle 连线能力保持可用；连线列表不再渲染。
