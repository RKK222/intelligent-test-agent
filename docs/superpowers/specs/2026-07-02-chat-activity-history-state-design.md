# 对话活动流与历史加载状态设计

## 目标

修复右侧对话面板中探索区不可折叠、文件写入重复且部分区域无法折叠、历史会话切换无加载反馈，以及任务终态后仍显示思考动画的问题；移除容易误解的 `read` 目录结果二次展示。

## 设计

- 文件活动只保留 `FigmaChatPanel` 已有的文件摘要渲染路径。`read`、`write`、`edit` 等文件工具不再进入通用工具详情，避免同一操作重复展示。
- 探索区复用现有 `expandedFileKeys` 状态，默认收起，用户点击标题行展开或收起读取路径列表。
- 不再展示 `read` 工具输出中的独立目录结果卡片；“F-COSS 目录 · 1 项”一类信息从对话活动流移除。文件内容结果仍保留已有折叠预览。
- 历史会话切换开始后立即展示“正在加载历史对话”状态。会话正文返回后立刻替换旧内容，消息反馈状态随后异步加载，不阻塞正文显示。
- 运行态以明确终态优先：`chatState.status` 或 `run.status` 任一已进入 `SUCCEEDED`、`FAILED`、`CANCELLED` 等终态时停止思考动画；启动请求仍未返回时继续保持 busy。
- 不恢复对话内文件 Diff 展示，不修改右侧面板拖拽宽度。

## 测试

- 组件测试覆盖探索区默认收起及点击展开/收起。
- 组件测试覆盖文件写入只渲染一份可折叠摘要，不出现通用工具详情。
- 组件测试覆盖 `read` 目录输出不再展示独立目录卡片。
- 工作台测试覆盖历史切换加载态和正文优先于反馈请求显示。
- 工作台状态测试覆盖任一明确终态立即停止运行动画。
