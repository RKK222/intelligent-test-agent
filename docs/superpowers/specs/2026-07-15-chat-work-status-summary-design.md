# 合并对话工作状态展示设计

> 2026-07-15 后续演进：完成摘要 actions 已改为透出当前 `work-status` 行；反馈按 `runId/runStatus` 评价整轮回复，每个成功历史 Run 保留入口。反馈数据设计详见 `2026-07-15-run-feedback-scope-design.md`，其中与本文“既有反馈 ID 不变”的描述不再适用。

## 目标

主智能体当前轮只展示一个工作状态块：运行中、重试、失败和停止时固定在原 Todo 面板位置、输入框上方，成功完成后移到该轮最后一个 assistant 输出下方并自动收为状态图标，与“满意 / 不满意”反馈按钮同排展示。文件修改块始终固定在输入框上方；历史轮在最后一个 assistant 输出下方收为单图标。问答、子智能体卡片和子智能体内部时间线保持现状。

## 数据与排序

- 根时间线新增 `work-status` 行，聚合 reasoning 以及除 `task`、`question` 外的全部 tool part 引用。
- 正文、retry、失败提示继续留在消息滚动区；最新非成功 `work-status` 与全部 `diff-summary` 由 `OpencodeTimeline` 投送到输入框上方的 Dock，成功完成的最新 `work-status` 留在时间线最后输出之后。
- 用户消息已进入时间线但尚无 assistant part 时，也生成状态块并显示真实轮次运行状态；没有事件时只渲染思考行，不保留空事件行。
- reducer 按 user message ID 保存 Todo 快照；新用户消息进入前归档上一轮 Todo 并立即清空当前 Todo，`todo.updated` 与 `todowrite` part 更新当前轮快照，历史持久消息按轮恢复最后一次 todowrite。
- 状态数据按轮保留；新用户轮次出现时，旧轮摘要在最后一个 assistant 输出下方收为单图标。
- 子智能体视图继续使用既有 reasoning/context/tool 行，避免改变子智能体展示方式。

## 展示与交互

- 细边框包裹状态内容，左侧竖向 `ShimmerDivider` 贯穿块高。
- 第一行保持现有“思考状态”标题、摘要、状态和 reasoning 展开样式；没有 reasoning 时不生成虚构正文。
- 事件首次出现后才增加事件行，并按首次出现顺序展示探索、技能、命令行、编辑、写入、补丁、网页、待办和未知工具类别。单次只显示图标，多次显示图标与数字，未出现类别不占位。因此初始状态只有思考一行，有事件后为两行。
- 当前轮有 Todo 时再增加 Todo 行，复用现有任务统计和展开列表；Todo 不依赖空事件占位，历史展开优先显示该轮快照，无法恢复时不展示错误任务。
- 点击事件图标打开与对话内容区等宽的悬浮气泡；气泡按可用空间选择上方或下方，最大高度为 `min(360px, 50vh)`，内部滚动并复用现有工具详情组件。
- 全时间线同时只允许一个事件气泡；再次点击、点击外部或按 Esc 关闭。历史状态同时只允许展开一个，点击图标原位展开，再次点击收起；新一轮开始关闭所有旧详情。
- 最新轮进入成功完成态后，状态从输入框 Dock 移到该轮最后一个 assistant 输出下方并显示紧凑摘要行：状态图标、“满意”和“不满意”按钮从左到右并列。反馈入口只在既有反馈资格成立时出现，不改变反馈 ID、选中态、提交中状态或不满意原因弹窗。
- 完成态摘要行中的状态图标是展开开关，带明确 ARIA 标签和展开状态。点击后保留摘要行，并在其下方原位展开完整状态块；文件修改块不随状态移动，继续留在输入框上方。再次点击图标收起详情，新用户消息进入时恢复默认收起。
- 仅 `completed` 成功态自动收起；失败、停止和重试保持完整状态，便于直接查看诊断信息。最新运行轮播放竖向流光，完成详情显示静态渐变线；收起摘要不保留完整状态块高度。

## 兼容边界

- `TimelineRow.work-status` 携带该轮 Todo 快照；运行态和对话投影输入增加可选的 `todoSnapshotsByUserMessageId`。
- `OpencodeTimeline` 接受可选 Dock 目标，并提供与完成态摘要同排的通用 actions 插槽；Dock 承载文件修改块和非成功的最新工作状态，完成摘要及 actions 留在时间线。`agent-web` 通过该插槽提供反馈按钮，`agent-chat` 不依赖反馈 DTO 或提交逻辑；未提供 Dock 或 actions 时仍能独立渲染。
- `ShimmerDivider` 增加向后兼容的 `orientation` 与 `animated` 可选属性，默认横向动画行为不变。
- 不改变 `AgentMessage`、`MessagePart`、HTTP API、RunEvent、数据库、权限或安全契约。

## 验收

- reducer/投影测试覆盖分轮 Todo 归档与恢复、空 assistant、事件聚合、计数、未知工具、状态与文件修改排序、多轮保留、retry/error、问答/子智能体不变。
- 组件测试覆盖初始单行、事件两行、Todo 附加行、非成功状态 Dock 投送、成功状态回到最后输出、Todo 展开、完成摘要 actions 插槽、图标展开/收起、Diff 固定 Dock、历史单图标及互斥展开、全宽气泡互斥与关闭和流光生命周期。
- 应用测试覆盖完成摘要与反馈紧跟最后 assistant 输出、Diff 仍在输入区、满意提交、不满意弹窗、选中/提交中状态、运行中和非成功终态隐藏，以及子智能体视图不展示主对话完成摘要。
- `ShimmerDivider` 测试覆盖纵向、静态模式和默认横向兼容。
- 执行前端 typecheck、Vitest、生产 build、`git diff --check`，并检查桌面与窄屏真实页面。
