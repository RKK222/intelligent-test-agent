# 历史工作状态展开交互统一设计

## 背景

同一会话中，最后一轮完成状态使用“固定摘要行 + 下方状态块”的结构：展开按钮始终可见，用户再次点击同一按钮即可收起。非最后一轮的历史工作状态目前使用两个互斥模板，展开后原按钮消失并改成状态块右上角的独立收起按钮，导致同类交互反馈不一致。

## 目标

- 历史工作状态的展开、收起交互与最后一轮完成状态保持一致。
- 点击“展开历史工作状态”后，原状态按钮不消失，状态块显示在按钮摘要行下方。
- 再次点击同一按钮隐藏状态块。
- 反馈入口继续与状态按钮位于同一摘要行。
- 保持一次只能展开一个历史工作状态的现有约束。

## 方案

仅调整 `TimelineRow.vue` 中历史 `work-status` 行的渲染结构，不改变时间线投影、状态归属或展开状态管理：

1. 历史行始终渲染一个纵向容器。
2. 容器第一行固定渲染 `Activity` 状态按钮和既有反馈插槽。
3. 按钮根据 `historicalWorkStatusExpanded` 动态设置“展开历史工作状态”或“收起历史工作状态”，并同步 `aria-expanded`。
4. 仅在展开时于摘要行下方渲染 `WorkStatusRow`。
5. 删除历史状态块右上角的独立 `ChevronUp` 收起入口及其专用样式。

`OpencodeTimeline.vue` 继续用 `expandedHistoricalStatusKey` 保存唯一展开行，因此切换到另一历史轮、新用户消息进入或当前行消失时的既有收敛行为不变。

## 测试

在 `opencode-timeline.test.ts` 中以用户交互覆盖：

- 初始只显示历史状态按钮，不显示对应状态块。
- 点击后同一个历史按钮仍存在，文案切换为“收起历史工作状态”，`aria-expanded=true`，状态块位于摘要行之后。
- 再次点击同一按钮后状态块隐藏，按钮恢复“展开历史工作状态”和 `aria-expanded=false`。
- 多个历史轮仍只允许一个状态块展开。
