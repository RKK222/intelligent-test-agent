# Mermaid 真菱形与多端口设计

## 目标与范围

改进 Flowchart/Graph 可视化编辑器的节点外形与连线体验：判断节点使用宽度大于高度的真实菱形轮廓，所有节点按照图方向提供 3 个入口和 3 个出口，使多条连线不再挤在同一个连接点上。

本次只修改 Mermaid 可视化层及其测试和稳定说明文档。Markdown 仍是唯一事实源；不修改 Mermaid 领域模型、parser、serializer、保存链路、后端 API、事件、数据库或全局状态。

## 节点外形

判断节点使用水平菱形轮廓，宽度大于高度。节点根元素和文字保持水平，不再通过旋转整个正方形来模拟菱形。轮廓由 CSS 多边形绘制，外层负责边框和选中态，内层负责背景色；节点内容位于安全内区，避免文字靠近菱形尖角。

右侧节点图形库中的“判断”预览使用同一水平菱形比例和轮廓方案，保证创建前预览与画布结果一致。矩形、圆角、胶囊和圆形节点外形保持不变。

## 方向感知端口

每个画布节点固定渲染 3 个入口 Handle 和 3 个出口 Handle。三个 Handle 沿所在边按 25%、50%、75% 均匀分布，并使用稳定 ID：入口为 `target-0`、`target-1`、`target-2`，出口为 `source-0`、`source-1`、`source-2`。

端口所在边跟随图方向：

- 从上到下：入口在顶部，出口在底部。
- 从下到上：入口在底部，出口在顶部。
- 从左到右：入口在左侧，出口在右侧。
- 从右到左：入口在右侧，出口在左侧。

判断节点的三个端口沿对应半边轮廓分布：中间端口落在入口或出口尖端，两侧端口落在相邻斜边，避免端口悬空在菱形外接矩形上。Handle 使用可见尺寸和悬停反馈，节点选中态不遮挡端口。

## 连线分配

Vue Flow 适配层按照领域边在草稿中的稳定顺序，分别统计每个起点的出边和每个终点的入边。第 N 条出边绑定到 `source-(N % 3)`，第 N 条入边绑定到 `target-(N % 3)`。因此，同一节点的前三条入边或出边会分配到三个不同端口；超过三条时按顺序循环复用。

新增、删除或交换连线后，适配层会根据最新边列表重新计算分配。端口 ID 只属于可视化投影，不写入领域模型或 Mermaid 文本。用户从任一出口拖到任一入口创建连线时，既有创建逻辑仍只记录起点和终点；下一次投影根据稳定规则统一分配端口，从而保持 Markdown 兼容性和确定性。

## 组件边界

端口方向、位置和 ID 生成放在可独立测试的纯函数中，由 `MermaidFlowNode.vue` 负责渲染六个 Handle，由 `vue-flow-adapter.ts` 负责为 Vue Flow edge 指定起点和终点 Handle。`MermaidVisualEditor.vue` 只同步图形库中的判断节点预览，不承担端口分配逻辑。

无效或未知图方向沿用现有从上到下的默认行为。端口分配不引入运行时错误分支，也不改变节点删除时同步删除关联边的现有行为。

## 测试与验收

测试覆盖以下行为：

- 判断节点和图形库预览使用宽度大于高度的水平菱形，文字不再反向旋转。
- 每个节点渲染 3 个入口 Handle 和 3 个出口 Handle，并带稳定 ID。
- 四种图方向下，入口和出口分别位于约定边，三个端口按 25%、50%、75% 分布。
- 同一节点的多条入边和出边依次绑定三个端口，第四条开始循环复用。
- 新增和删除连线后端口分配保持确定，Mermaid 序列化结果不包含端口私有信息。
- 既有节点选择、属性编辑、拖拽创建节点和 Handle 创建连线行为继续通过回归测试。

完成后执行 editor 定向 Vitest、editor 类型检查、前端全量测试、生产构建和 `git diff --check`，并在本地浏览器验证真菱形比例、六个端口、四种图方向与多条连线的可读性。
