# 快速开始

本页适合第一次进入 MIMO 测试智能体的用户。推荐按下面顺序完成准备，不要跳过权限、SSH、应用或工作区状态检查。

## 先记住四个入口

1. **应用**：顶部应用下拉只负责切换和加入已有应用；普通用户不能在这里新建应用。
2. **工作区**：顶部选中应用后，还要点击左下角的 workspace/version 切换按钮，真正选中一个 workspace 和版本，文件树才会有内容。
3. **对话**：不需要先创建会话，输入第一条任务并发送，系统会自动建立主对话；“新建对话”用于主动开始另一条草稿。
4. **需求子条目**：选中 workspace 后，点击“工作空间”标题右侧的小地球，在外部需求页面选择并引入子条目；返回后再在对话框用 `#` 选择它。

## 1. 确认权限并配置 SSH

先从右上角头像确认角色。超级管理员可以新建应用；应用管理员可以维护已有应用的成员、版本库和工作空间；普通用户可以配置自己的 SSH Key、加入应用并使用个人工作区。

需要访问 Git 时，打开“设置 → 个人设置”，添加已经在公司 Git 服务登记过公钥的 SSH 私钥。平台只回显名称和指纹。完整准备顺序见[首次使用前准备](./first-time-setup.md)。

## 2. 找到并确认当前应用

查看顶部应用下拉框，确认当前应用就是要开展测试工作的应用。切换应用后，版本、个人工作区、文件树和历史对话都会随应用变化。

普通用户如果目标应用不在顶部列表，点击“加入其他应用”，选择已授权的应用后保存。普通用户看不到“新建应用”不是页面缺失：应用由有权限的管理员在“设置 → 应用管理”创建，再把成员加入应用。创建应用的完整管理步骤见[首次使用前准备](./first-time-setup.md)。

## 3. 先选中 workspace，再看文件树

顶部选中应用后，点击左下角双向箭头的“切换工作空间”按钮：先在一级菜单选择工作空间模板，再在右侧二级菜单选择版本。按钮最终显示“工作空间 / 版本”时，才算真正选中 workspace；页面会随后准备当前用户的 `default` 个人工作区并加载左侧文件树。

如果左侧“工作空间”一直空白，先检查这三点：是否只选了应用而没有选 workspace/version；是否还在等待个人工作区准备；该版本是否已由管理员创建运行态工作区。详细说明见[应用版本与个人工作区](./workspace.md)。

## 4. 准备 TestAgent 专属进程

首次登录或进程已停止时，页面会显示“分配专属进程”“启动进程”或“初始化进程”。小宠物也会主动提示。

点击按钮后等待步骤窗口完成。成功的判断标准是进程状态变为可用，对话输入区不再提示先初始化。详细说明见[初始化 TestAgent 进程](./process-initialization.md)。

## 5. 建立第一条对话并引入子条目

进程和工作区均可用后，可以直接在右侧输入任务。第一条消息发送时系统会自动创建主对话，不需要先点击“新建对话”；右侧“新建对话”按钮只用于清空当前草稿、开始另一条对话。

建议任务描述至少包含：测试对象、预期动作、输入材料和期望产物。例如：

> 阅读当前需求子条目和详细设计，生成接口测试案例，并把最终案例保存到 041-测试设计目录。

选中 workspace 后，点击左侧“工作空间”标题右侧的小地球，在外部需求页面选择要引入的子条目并保存。返回工作台后文件树会刷新；在输入框键入 `#`，选择对应需求子条目，平台会把需求、设计、编码和测试阶段的相关文件一起加入本轮上下文。也可以使用 `@` 添加 Agent 或单个工作区文件。详细说明见[对话与上下文](./conversation.md)。

## 6. 在哪里查看结果

- 对话区：查看 Agent 的回答、工具过程、提问卡和权限确认。
- 中央编辑区：查看和编辑工作区文件、Markdown 预览与差异。
- 左侧 Git 变更：查看个人 worktree 的真实变更，按需暂存、本地提交或把允许发布的文件投影并推送到应用 feature 分支；`spec/**` 始终只做个人本地提交。
- 底部运行与终端：查看 Run 状态，或在已有会话上下文时连接终端。

::: tip 不确定按钮作用时
点击顶部问号打开本手册；如果按钮旁有“查看操作手册”，会直接定位到对应章节。无需先建立主对话：工作区和 TestAgent 服务就绪后，可以在帮助中心或直接点击小宠物，依据用户手册提问。
:::
