# 设置与权限内操作

设置弹窗会根据当前账号角色显示不同入口。普通用户先看“个人设置”；如果账号同时拥有应用管理员权限，还会看到应用和版本库相关管理入口。页面不会把超级管理员专属的“用户管理”作为普通用户的操作步骤。

## 设置面板怎么用

打开工作台左侧活动栏的齿轮后，按下面顺序操作：

1. 普通用户进入“个人设置 → SSH Key”，填写 Key 名称、粘贴私钥并点击“添加 SSH key”；删除旧 Key 时点击对应的“删除”。
2. 应用管理员先进入“应用管理”，在顶部下拉框选中目标应用，再依次处理应用人员、版本库关联和工作空间；如果没有可关联版本库，先进入“版本库管理”新增。
3. 工作空间保存成功后，回到工作台左下角选择 workspace 和版本；只选应用、不选 workspace/version 时，文件树保持空白是正常的。

下面的三类配置对应这条操作链：用户配置负责“谁能进入应用”，版本库配置负责“应用使用哪些代码库”，应用工作区配置负责“从哪个测试工作库、分支和目录创建 workspace”。

## 普通用户：个人设置

普通用户打开工作台左侧活动栏的齿轮，进入“设置 → 个人设置”，主要完成自己的 SSH Key 管理：

1. 没有 Key 时，填写便于识别的 **SSH Key 名称**。
2. 粘贴完整的私钥内容（通常以 `-----BEGIN ... PRIVATE KEY-----` 开头）。
3. 点击“添加 SSH key”，看到名称和指纹后，说明保存成功。
4. 已添加的 Key 会显示名称和指纹；不再使用时点击“删除”。

平台不会回显私钥。公钥仍需提前登记到公司 Git 服务，并确认该账号有目标仓库和分支权限；设置里的 Key 只代表当前用户，不会借用管理员的 Key。SSH 失败时先回到这里检查 Key，再联系管理员确认 Git 权限。

## 账号同时有应用管理员权限时

应用管理员在设置中还会看到“应用管理”和“版本库管理”。这些入口用于为普通用户准备应用和工作区，不是普通用户自己创建应用的入口。

三类配置与页面入口的对应关系如下：

| 配置 | 页面入口 | 用途 |
| --- | --- | --- |
| 用户配置 | 应用管理 → 应用人员管理 | 把用户加入应用，或移除已有应用成员 |
| 版本库配置 | 版本库管理；应用管理 → 应用与版本库关联 | 登记 Git 版本库，并决定哪些版本库属于当前应用 |
| 应用工作区配置 | 应用管理 → 工作空间管理 | 从已关联的测试工作库、分支和目录创建应用工作空间 |

### 应用管理

先在顶部应用选择框选中目标应用，再按三个页签操作：

#### 用户配置：应用人员管理

1. 在“添加成员”输入用户 ID、用户名或统一认证号。
2. 从搜索结果中选中用户，点击“添加”；空输入不会查询用户库。
3. 移除已有成员时点击右侧删除按钮，并在页面内确认。

普通用户要进入应用，必须由管理员先在这里加入；这里加入的是应用成员，不会改变用户的全局角色。

#### 版本库与应用关联

1. 在“应用与版本库关联”选择已登记的版本库，点击“关联”。
2. 关联错误时点击“解除”并确认。
3. 没有可选版本库时，先到“版本库管理”完成版本库配置。

#### 应用工作区配置：工作空间管理

1. 选择已关联的测试工作库。
2. 选择分支、工作空间别名和目录；测试工作库只能选择应用同名目录下的一级子目录。
3. 点击“保存”，等待准备目录和创建运行态工作区的进度全部成功。
4. 回到工作台左下角，普通用户仍需选择 workspace 和版本，文件树才会加载。

应用管理员不能点击“新建应用”。如果顶部或设置中没有目标应用，应让超级管理员创建应用，再由应用管理员把成员加入。

### 版本库配置：版本库管理

“版本库管理”用于维护应用可关联的 Git 版本库：

1. 点击“新增”，填写部署模式、版本库地址、中文名称、英文名称和版本库类型。
2. 测试工作库才能用于创建应用工作空间；应用代码库和应用资产库按各自用途关联。
3. 已有版本库可点击“编辑”修改名称和英文名称；地址、部署模式和类型用于识别，不能在编辑时随意改变。
4. 新增或读取远端分支使用当前操作者自己的 SSH Key，先完成“个人设置”。

## 为什么我看不到某个设置入口？

设置菜单按权限显示：

| 账号权限 | 可见入口 | 普通使用重点 |
| --- | --- | --- |
| 普通用户 `USER` | 个人设置 | 配置或删除自己的 SSH Key，然后从顶部加入应用、从左下角选择 workspace/version |
| 应用管理员 `APP_ADMIN` | 应用管理、版本库管理、个人设置 | 为应用成员准备版本库、关联和工作空间，并维护自己的 SSH Key |

看不到“应用管理”或“版本库管理”通常不是页面故障，而是当前账号没有应用管理员权限。看不到“新建应用”也属于正常权限边界；普通用户应联系管理员处理应用和工作空间准备。
