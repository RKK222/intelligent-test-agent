# 首次使用前准备

第一次使用时，建议按“确认权限 → 配置 SSH → 准备应用 → 准备工作空间 → 初始化 TestAgent 服务”的顺序操作。SSH 和工作空间负责 Git 文件访问，TestAgent 专属进程负责执行 Agent；它们互相独立，但缺少任何一项都无法顺利完成带代码的任务。

## 先确认自己的角色

右上角头像菜单会显示当前角色。不同角色的稳定操作范围如下：

| 角色 | 可以做什么 | 不能做什么 |
| --- | --- | --- |
| 超级管理员 `SUPER_ADMIN` | 新建应用；管理用户和全局角色；管理公共 Git；继承应用配置管理权限；管理运行资源 | 仍需为自己的 Git 操作配置个人 SSH Key |
| 应用管理员 `APP_ADMIN` | 管理已有应用的成员、版本库关联和工作空间；维护并发布应用 Git 中的 Agent/Skill/rules/templates | 不能新建应用，不能管理平台用户、全局角色或公共 Git |
| 普通用户 `USER` | 配置自己的 SSH Key；加入已授权应用；使用个人 worktree；维护并发布 `docs/**`；本地提交 `spec/**`；发起对话 | 不能修改公共 Git 或应用 Agent/Skill/rules/templates，不能发布 `spec/**` |
| 系统管理员 `SYSTEM_ADMIN` | 当前版本保留的全局角色 | 当前设置页不会因此开放应用管理或超级管理员功能 |

角色不正确时，由超级管理员进入“设置 → 用户管理”调整。应用成员和全局角色是两件事：普通用户即使是某个应用成员，也不会获得应用配置或公共 Git 的写权限，但仍可在个人 worktree 中维护普通文件和 `docs/**`。

设置弹窗会按角色隐藏无权限入口。普通用户的设置操作、应用管理员如何准备成员和工作空间，见[设置与权限内操作](./settings.md)。

## 配置个人 SSH Key

Git 使用 SSH 地址时，每个实际执行 Git 操作的用户都要配置自己的私钥。平台不会共用其他用户的 Key。

1. 在受信任终端准备一对 SSH Key；没有 Key 时可使用 `ssh-keygen -t ed25519 -C "你的统一认证号"` 创建。
2. 把公钥（通常是 `.pub` 文件）登记到公司 Git 服务，并确认该账号对目标仓库和分支有权限。
3. 在本机先按公司 Git 服务要求验证 SSH 连通性；主机、端口和用户名以公司实际配置为准。
4. 打开工作台左侧活动栏的“设置”，进入“个人设置”。
5. 输入便于识别的 SSH Key 名称，并粘贴以 `-----BEGIN ... PRIVATE KEY-----` 开头的完整私钥，点击“添加 SSH key”。
6. 页面出现 Key 指纹后，关闭设置。平台只回显名称和指纹，不回显私钥。

::: warning 私钥安全
只粘贴自己的专用私钥，不要在对话、日志、截图或工单中发送私钥。浏览器会先加密私钥再提交，删除 Key 后，依赖该 Key 的远端分支、目录、clone、pull 和 push 都会失败。
:::

## 新建应用

新建应用是平台级主数据操作，只允许超级管理员执行。

1. 打开“设置 → 应用管理”。
2. 点击应用选择框右侧的“新建应用”。
3. 填写唯一的“应用 ID”和展示用“应用名称”。应用 ID 建议使用稳定业务编码，例如 `F-COSS`，创建后不要把它当作可随意修改的名称。
4. 点击“创建并启用”。创建成功后页面会自动选择新应用。
5. 在“应用人员管理”中搜索并添加应用管理员或普通成员。

应用管理员只能管理已存在的应用。如果看不到“新建应用”，先确认头像菜单显示的是“超级管理员”。

普通用户不需要也不能在工作台里新建应用。普通用户的动作是：在顶部应用下拉点击“加入其他应用”，选择管理员已经创建并授权的应用；如果列表里也没有目标应用，需要联系管理员先创建应用并添加成员。

## 新建应用工作空间

这一步由应用管理员或超级管理员完成，并且操作者必须先配置自己的 SSH Key。

1. 在“设置 → 版本库管理”新增版本库，填写部署模式、Git 地址、名称、英文名称和版本库类型。
2. 进入“设置 → 应用管理”，选择目标应用。
3. 在“应用与版本库关联”中选择刚才的版本库并点击“关联”。
4. 切换到“工作空间管理”，从已关联的测试工作库中选择版本库，再选择分支；其它版本库类型不会出现在创建下拉中。
5. 填写工作空间别名。
6. 在目录树中选择工作目录。测试工作库只能选择“应用同名根目录”的一级子目录，也可以先在该根目录下新增一级目录。
7. 点击“保存”，等待“准备目录、创建运行态工作区”等进度全部成功。
8. 回到工作台顶部选择应用，再在左侧选择应用版本和自己的个人工作区。

工作空间模板由管理员创建；普通用户第一次进入该版本时，页面会按提示创建或选择自己的 `default` 个人工作区。两者不要混淆。

对普通用户来说，最容易漏掉的是“选中 workspace”：顶部选中应用以后，还必须点击工作台左下角的“切换工作空间”按钮，选择工作空间模板和版本。只选应用、不选 workspace 时，左侧文件树保持空白是预期行为。

## 最后初始化 TestAgent 服务

完成上面准备后，再从小宠物或对话区提示点击“分配专属进程”“启动进程”或“初始化进程”。进程变为可用、应用和工作区都已选中后，才发送第一条依赖文件的任务。

如果任务不访问 Git，SSH Key 并不是进程启动本身的技术依赖；但首次完整配置仍建议按上述顺序完成，避免进程已就绪却在读取分支或创建工作空间时失败。
