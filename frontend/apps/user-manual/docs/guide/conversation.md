# 对话与上下文

右侧对话区用于发起测试任务、查看执行过程和处理 Agent 的交互请求。第一条消息发送时会自动创建主对话。

## 对话到底怎么建？

正常流程不需要先建会话：选中应用和 workspace、确认 TestAgent 服务可用后，在输入框写任务并点击发送，系统会在发送时自动创建主对话并把第一条消息作为标题来源。

右侧输入卡片的“新建对话”图标用于主动清空当前草稿、开始另一条对话。它不会删除历史对话，也不会取消后台已经运行的任务；历史对话仍可从历史入口找回。

## 怎样写清楚任务

一条可执行的任务通常包含四部分：

1. 测试对象：哪个需求、接口、模块或文件。
2. 动作：分析、设计案例、执行测试、修复问题或审核结果。
3. 约束：使用哪些材料、目录、格式或测试方法。
4. 产物：结果写到哪里，需要哪些文档或代码。

## 安排夜间执行

算力紧张时，可以把任务安排在北京时间晚上 9 点到次日早上 7 点异步启动：

1. 先在当前消息框写好任务，并按需添加 Agent、模型、文件或需求子条目。
2. 点击发送按钮左侧的定时图标。
3. 在 15 分钟启动时段中选择一个时间。系统会根据全局名额自动标出推荐时段；显示“已满”的时段不能选择。
4. 确认后等待“已安排”提示。只有提交成功后，草稿和上下文附件才会清空。

当前已经打开历史对话时，任务会在该对话继续；处于全新空白对话时，系统会为任务预留一条新对话。弹层不再单独询问执行位置。

对话顶部的“待执行任务”页签会分页收齐并展示自己所有尚未启动的夜间任务；切换页签会立即刷新，当前对话的任务排在最前，其余按启动时段排序。任务卡会显示内容预览、启动时段、创建时间、状态以及发生过的顺延次数；任务仍为“待执行”时可以改期或取消，进入“正在启动”后不能再改期或取消。取消前需要在卡片内再次确认。

已有对话安排夜间任务后，在任务取消、最终失败或成功启动前，当前对话不能继续发送普通消息，也不能归档；“新建对话”仍可使用，可以先处理别的工作。任务启动后直接复用普通对话的执行过程和结果展示，用户消息旁会显示“夜间定时执行”和北京时间的实际启动时间。如果窗口内始终无法启动，当前对话会显示失败卡；关闭失败卡不会重新执行任务。

## 添加上下文

- `@Agent`：选择当前运行态已经加载的 Agent。候选项以英文名为主、中文名为辅，选择后仍插入稳定的 Agent 技术 ID。
- `/Skill`：选择当前 Agent 可用的 Skill。候选项同样以英文名为主、中文名为辅，选择后仍插入稳定的 Skill 技术 ID。
- `@文件`：选择工作区文件并作为显式上下文加入本轮消息。
- `#需求子条目`：只解析 `spec/<需求项>/<阶段>/<子条目>` 结构，按同名子条目聚合需求、设计、编码和测试阶段文件；工作区根目录下的旧需求结构不会进入候选。
- 小地球引入：先在左侧“工作空间”标题右侧点击小地球，从外部需求页面引入子条目；返回后再用 `#` 选择，避免只把单个阶段文件漏带进任务。
- 编辑器选区：把当前选中的代码或文本加入下一轮消息。
- Git hunk：从差异视图引用当前变更片段。

附件较多时页面只折叠展示，不会减少实际发送的文件。发送前仍会执行二进制、重复和容量检查。

## 处理中可以做什么

- 当前 Run 忙碌时继续发送的 Prompt 会进入跟进队列。
- Agent 出现权限卡时，按本次允许、始终允许或拒绝作出选择。
- Agent 出现提问卡时填写答案，提交后原卡片会收敛为完成态。
- 需要终止时使用停止按钮；只有 Run 已创建并处于运行态时可以终止。

## 小宠物旁路问答

主对话至少发送过一条消息后，点击小宠物会基于当前主 Session 创建临时上下文，不写入主对话历史，适合追问“刚才做了什么”“为什么失败”。

小宠物提供侦探犬、雷达兔、星探狐、巡检小鸟和数据刺猬五种图片形象。打开宠物浮层后，点击标题栏的脚印按钮可以选择显示方式：

- 每日轮换：按本地日期依次更换值班伙伴。
- 每天随机：每天抽取一个伙伴，当天保持不变。
- 自主选择：固定使用自己选中的伙伴。

在同一处可以用“宠物大小”滑杆调整显示比例（75%–150%）。比例保存在当前浏览器，旧版本没有该设置时按 100% 显示。

显示偏好只保存在当前浏览器，不会影响旁路问答内容，也不会同步到其他设备。

帮助中心中的“小宠物问答”会携带当前手册章节，适合询问按钮含义和操作顺序。没有主对话时，只要已经选择工作区且 TestAgent 服务可用，小宠物就会创建独立的内部手册问答 Run；它不会创建普通聊天会话，也不会把问题或答案写入主对话历史。

## 历史对话

历史列表可以搜索和切换已有 Session。切换尚未完成时不要重复发送消息；只读历史会明确显示原因。新建对话只清理当前草稿，不会删除历史 Session。
