# 初始化 TestAgent 进程

TestAgent 专属进程负责执行对话、Agent、Skill 和工具调用。它与应用工作区不是同一个概念：进程提供执行能力，工作区提供文件和 Git 上下文。

首次完整使用前，先确认角色、个人 SSH Key、应用成员关系和工作空间已经准备好，具体步骤见[首次使用前准备](./first-time-setup.md)。SSH 并非进程启动本身的技术依赖，但缺少 SSH 会让后续分支、目录、clone、pull 和 push 失败。

## 先看懂按钮文案

| 页面文案 | 含义 | 该怎么做 |
| --- | --- | --- |
| 分配专属进程 | 当前用户还没有绑定可用进程 | 点击后等待平台选择服务器和容器并启动进程 |
| 启动进程 | 已有服务器或端口归属，但进程没有运行 | 点击后等待原位置重新启动并完成健康检查 |
| 初始化进程 | 状态信息不完整或需要重新建立运行环境 | 点击后按步骤窗口完成初始化 |
| 初始化中 / 启动中 | 后台操作仍在进行 | 不要重复点击，等待步骤窗口给出成功或失败结果 |

## 正常操作步骤

1. 点击对话状态卡或小宠物提示中的初始化按钮。
2. 在启动步骤窗口中等待分配、启动、状态登记和健康检查完成。
3. 窗口显示“已完成”后关闭窗口。
4. 确认页面状态变为可用，再发送任务。

平台只有在进程管理状态、PID 和 HTTP 健康检查都通过后，才会把进程视为可用。按钮暂时不可点击通常表示仍在检查或初始化，不需要刷新页面重复提交。

## 常见失败怎么处理

### 提示公共配置尚未初始化

普通用户不能自行修复。请把错误提示和 `traceId` 发给超级管理员。管理员进入“系统管理 → 配置管理 → TestAgent 公共配置管理”，在承载该进程的服务器上初始化公共配置后再重试。

### 提示没有可用容器或 manager 未连接

联系平台管理员检查运行管理中的容器状态、端口容量和 manager WebSocket 连接。恢复后重新点击初始化即可，不要手工填写服务器地址。

### 长时间停留在初始化中

先保留步骤窗口中的 operationId、错误码和 traceId，再联系管理员检查目标服务器日志。不要连续刷新或反复点击，以免同时发起多次操作。

### 进程可用但仍不能发送

继续检查是否已经选择应用版本和个人工作区。进程可用只代表执行能力正常，不代表当前对话已经有文件工作区。

## 与其他“初始化”的区别

- TestAgent 进程初始化：普通用户通常可以从对话状态卡发起。
- 应用版本/个人工作区：在左侧工作区或设置中选择、创建，决定文件和 Git 上下文。
- 应用 Agent 配置包：为应用工作区创建 `.opencode` 配置内容，不负责启动进程。
- 公共 Agent 配置仓库：由超级管理员按服务器维护，普通用户只读使用。
