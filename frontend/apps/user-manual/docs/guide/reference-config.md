# 引用配置

“引用配置”把应用资产库中的规格、文档等目录加入当前个人工作区的 OpenCode 配置。资产库会先同步到各服务器的只读引用目录，页面再把选中的目录写入当前个人工作区 `.opencode/opencode.jsonc`；它不会把资产文件复制进个人 Git。

## 使用前确认

- 当前角色是应用管理员或超级管理员。超级管理员继承应用管理员权限；普通成员看不到入口。
- 顶部已经选择目标应用，左侧已经进入该应用版本下自己的个人工作区，并且工作台已加载对应运行态工作区。
- 目标代码库已经在“设置 → 应用管理 → 应用与版本库关联”中关联到当前应用，版本库类型为“应用资产库”。
- 首次初始化或同步需要读取远端 Git；操作者应先在“设置 → 个人设置”配置自己的 SSH Key，并确保该账号有目标仓库和分支读取权限。

缺少应用、个人工作区或运行态工作区时，工作台不会显示“引用配置”按钮。先完成[应用版本与个人工作区](./workspace.md)中的选择步骤，不要在空工作区手工编辑物理路径。

## 初始化或同步资产库

1. 在工作台底部找到工作空间切换按钮，点击它后面的“引用配置”图标。
2. 在弹窗左栏选择一个应用资产库。
3. 如果资产库尚未初始化，打开分支选择，选择要长期使用的分支后点击“初始化”。初始化会固定该分支和当时的远端提交，并开始为在线服务器准备副本。
4. 如果资产库已经初始化，选择它会在原分支上发起同步，平台重新读取远端最新提交。页面每 2 秒刷新一次活动状态，不要连续重复点击。
5. 等待状态变为“已就绪”。如果某台服务器显示 `DEFERRED`，表示该服务器当前离线；它恢复后会自动补同步，不影响其它在线服务器先完成。当前工作区所在服务器的副本未就绪时，右侧目录树暂不可用。

::: warning 初始化后的固定字段
首次初始化成功后不能切换分支，也不能修改该版本库的英文名或把类型改成其它版本库类型。英文名同时是每台服务器的引用目录名；如初始化前填错，应先修正版本库配置，再执行初始化。
:::

## 选择橙色目录

资产库就绪后，右栏显示当前服务器副本的目录树。可以展开目录查看内容，但只有仓库根层显示为橙色的目录可以选作引用；橙色名称由平台的 SDD 目录清单决定，默认包括 `docs` 和 `spec`。文件、嵌套目录、`.git` 和符号链接都不能选。

点击橙色目录后，页面生成以下只读字段：

- 参考别名：`{目录名}-{版本库英文名}`。
- 路径：`{env:OPENCODE_REFERENCES_DIR}/{版本库英文名}/{目录名}`。
- 规格驱动目录名称：当前橙色目录名。

可以选择“是否合并”，并填写必填的“描述”。描述应说明这份资料的用途和适用范围，方便 Agent 判断何时引用。

## 保存或更新

- 目标别名还不存在时，按钮显示“保存”。
- 已存在相同别名、相同 path 的本地引用时，页面加载现有 `merge` 和描述；只有修改后“更新”按钮才可用。
- 点击保存或更新前，页面会重新读取最新 `.opencode/opencode.jsonc`，只补丁目标 `references` 节点，并保留其它配置、注释、尾逗号和未知字段。读取和写入都通过平台工作区文件通道完成。
- 既有别名是字符串、Git 引用对象或指向其它 path 时，平台不会猜测覆盖；请先人工检查同名配置并处理冲突。

保存成功后，该修改属于当前个人 worktree。按团队流程在 Git 变更中检查、暂存和提交；`.opencode/**` 的发布仍只允许应用管理员或超级管理员。

## 什么时候对进程生效

引用资产同步完成和配置保存都不会自动重启当前正在运行的 TestAgent 专属进程。`OPENCODE_REFERENCES_DIR` 环境变量只会在用户进程下次新启动，或管理员通过平台执行受管重启时注入。已有进程可以继续当前任务；需要马上使用新引用且当前进程启动时还没有该变量时，请在没有运行中任务后通过平台受管重启，或等待下次启动，不要手工杀进程或修改 manager 环境。

## 错误处理

### 分支、SSH 或 Git 错误

确认当前用户 SSH Key 已配置、Git 账号有仓库和分支读取权限，且分支仍存在。初始化后不能改分支；需要换分支时联系平台管理员评估重新配置，不要直接修改服务器目录。

### 一直同步中、重试中或显示失败

短暂网络/超时会自动退避重试。目录不是可接管 Git 仓库、本地有未提交修改、origin/分支不一致、提交发生分叉、参数或凭据错误会阻塞本轮。不要删除或覆盖服务器目录；把版本库、服务器状态、页面错误码和 `traceId` 发给管理员排查，修复后再点击同步开启新一轮。

### 目录树不可用

先确认总体状态和当前服务器副本都已就绪。路径越界、`.git`、符号链接和不存在目录会被后端拒绝；页面只允许从树中选择，不要自行拼接绝对路径。

### 保存配置失败

确认仍在原应用和个人工作区，且自己仍有应用管理员权限。JSONC 语法错误、`references` 不是对象、同名别名类型或 path 冲突时，页面会保留原文件并显示错误。修复文件后重新选择目录；如错误带 `traceId`，联系管理员时一并提供，但不要发送 SSH 私钥、token 或完整敏感配置。
