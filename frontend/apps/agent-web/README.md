# @test-agent/agent-web

## 工程定位

Vue 3 + Vite SPA 主应用，组合 Web IDE 工作台、文件树、Monaco 编辑器、Agent 对话、Run 面板和 Diff 查看。

## 主要职责

- 提供 `/login` 登录页、`/` 工作台页面和 `/s/[sessionId]` 只读 transcript 页面（vue-router 客户端路由），未知路径回退到工作台。
- 组合 dockview-vue 三栏布局和底部运行面板。
- 组合 Figma Web IDE 风格的 40px 顶栏、48px activity rail、紧凑左侧文件面板、中间编辑器和右侧 Agent 面板；Run/Terminal 默认隐藏在底部抽屉中，通过 activity rail 打开。
- 顶栏 MIMO 小宠物使用低饱和蓝灰主体与青色、紫色点缀；右上角按钮使用与宠物相同的天线、脑袋和眼睛几何，可立即唤起或收起，连续 60 秒无鼠标、键盘、滚动或点击且页面聚焦可见时也会自动出现。宠物支持指针拖动或键盘重新定位，位置仅保存在浏览器本地，保存坐标只作为下一次出现的起点；手动安放后立即恢复自然随机动作与自然离场计时，并额外支持快速抖动和旋转庆祝。右侧 TestAgent 进程状态在无拖动偏好时保留聊天顶部内联卡片；用户收起或有效拖动后使用可拖动的 8px 半透明柔光点，浮动位置仅以 `{ v: 2, cardX, cardY, dotSide }` 持久化：卡片锚点是唯一坐标，`dotSide` 只记录绿点相对卡片的方向，绿点由二者、卡片尺寸和视口边距派生。旧 `{x,y}` 绿点偏好会在读取时迁移为 v2 并保留其视觉方位；已校验 v2 记录会等展开卡片取得真实尺寸后才夹取和改写。窗口缩放和卡片内容变高只夹取并保存同一锚点。子 Agent 视图不显示两种进程状态入口；这些交互不新增或变更 API、RunEvent SSE、数据库、安全或兼容性契约。
- 通过 `@tanstack/vue-query` 调用 `@test-agent/backend-api`。
- 应用和个人工作区切换只使用后端返回的运行态 Workspace id；普通前端不传物理目录创建 Workspace，切换时清空旧文件树、编辑器、Diff、Session/Run、队列和聊天运行态。
- 通过 `@test-agent/event-stream-client` 订阅 RunEvent SSE；事件只作用于当前活动 Run，旧订阅晚到的 `run.cancelled/run.succeeded/run.failed` 不会覆盖下一轮运行态。`session.status.retry` 会保持当前 Run 为运行中，在时间线展示错误内容和 60 秒倒计时；第 1/2 次倒计时到期后前端会用最近一次 Run 草稿自动重试，第 3 次倒计时结束仍未恢复时本地标记对话失败。
- 订阅既有 wire name `session.updated`：只消费后端在平台标题成功持久化后追加的 `platformSessionTitleSynchronized: true` 与 `platformSessionTitle`，不再从 `info` 或 `rawPayload` 推断标题。仅当事件所属订阅 Run 的平台 Session 仍为当前会话、事件不是 child session 时，更新当前会话和已加载历史会话的标题，并失效 sessions cache；切换历史会话后到达的旧订阅事件，以及没有平台确认标记的 child、空标题、不匹配或持久化失败事件，都不覆盖当前标题。该消费不新增 API、DTO、事件 wire name、数据库/Flyway 或 SDK，旧客户端可忽略该既有事件。
- 通过 `@test-agent/event-stream-client` 的用户级运行态 fetch SSE 订阅历史会话运行中计数和 `question.asked` 待回答提醒；运行中点击“新建对话”只清空当前视图并关闭当前 RunEvent SSE，不调用 cancel/abort，后端继续接收 opencode 事件，历史切回时仍使用 session-tree/messages 与 active-run 恢复。
- 在 `AgentWorkbench` 内维护会话级原始报文内存缓存，右侧 `FigmaChatPanel` 的“原始输出”浮层展示当前页面生命周期内捕获的会话相关 HTTP 请求/响应正文和 RunEvent SSE `MessageEvent.data`，并支持按请求/响应/SSE 类型和本地关键词搜索过滤标题、URL、traceId、runId 与正文，命中的可见文本会高亮。该能力仅用于前端调试，不新增后端 API、表结构或 RunEvent 契约；浏览器 `EventSource` 不暴露完整 HTTP 字节流，因此 SSE 只保存前端实际收到的 `data` 原文。若 EventSource 报告连接异常但后端尚未发出 `run.failed/run.succeeded/run.cancelled`，前端会在原始输出中追加一条本地 SSE 诊断记录，并在时间线展示一次连接异常提示，等待浏览器自动重连。
- 用 Pinia 工作台状态管理打开文件、活动 tab 和 Diff 选择。
- 承载"实时追踪"行为：用户开启后，运行中的 `message.part.updated`/`diff.proposed` 会打开最近变化文件、刷新只读预览，并把 `files[].additions/deletions` 传给文件树展示行数。所有 `diff.proposed` 都会立即强制刷新变更文件的父目录和工作区 Git Diff，即使 Run 尚未结束或实时追踪关闭，新建/删除文件也无需刷新整页即可出现在文件树和 Git Changes。`workbench-utils.inferDiffFromToolPart` 在 opencode 1.17.8 `session.diff` 事件 diff 字段为空时，基于 `write`/`edit`/`apply_patch`/`str_replace`/`multi_edit`/`create_file` 工具的 input 估算新增/删除行数并补齐 `RunDiffFile`，让"文件变更卡片 +N"在写盘后即时刷新；该推断与"实时追踪 toggle"解耦，关闭实时追踪时仍会同步 diffFiles 和文件树，但不再打开只读预览 tab。
- 组合用户级 Session History 远端搜索、分页加载和切换；右侧历史抽屉默认展示当前登录用户最近 30 条 ACTIVE 会话，按 `updatedAt desc` 排序，加载更多每次追加 30 条，并展示标题、应用名、工作空间名称、版本、创建时间和最后更新时间。历史按钮会展示运行中计数、旋转图标和待回答铃铛，历史卡片会按用户级运行态摘要显示运行中 Spinner、已完成图标和待答铃铛。历史列表不再拼接“本地会话/当前 Run”伪记录，`pinned` 字段只保留兼容展示，不参与排序。
- 右上角应用菜单只展示当前用户已加入的应用；未加入应用只出现在“加入其他应用”弹窗，管理员也需要先通过加入入口成为成员后才能在切换菜单里看到。加入成功后只刷新已加入/可加入应用列表，不自动切换到新应用。切换应用后优先按应用最近使用记录只读加载已有 default 私人工作区，没有可用记录时清空旧文件树并提示选择工作空间；应用名旁边提供用户头像菜单，可退出登录并切换用户，头像菜单每次打开都会实时查询 `/api/internal/agent/opencode/processes/me` 并用灰/绿/红展示“未分配 / 运行中(服务器名 / 当前地址) / 未运行(服务器名 / 当前地址)”，当前地址缺失时只展示服务器名；若状态查询失败且没有可用响应，头像菜单显示“状态未知”而不是误判为未分配。未选择应用时不自动读取普通工作区文件树，避免应用与左侧工作区错位；下拉菜单顶部以灰显行展示当前用户的角色中文名（来自 `/api/auth/me` 的 `roleLabels`，链路 `users` → `user_roles` → `dictionaries.dict_label`），缺失时整行不渲染。
- 应用目录加载完成且 `selectedAppId` 为空时，前端 `trySelectDefaultApp` 先等 `GET /recent-workspace`（`WorkspaceRuntimeResponse.appId` / `versionId` / `applicationWorkspaceId` 仅在最近工作区接口中填充）返回上次的全局最近工作区并解析出 `appId`，再以该 `appId` 调用 `handleSelectApp`。应用级 recent 命中且带 `versionId` 时，`pickDefaultWorkspaceForApp` 只调用 `GET /workspace-versions/{versionId}/personal-workspaces`，选择 `workspaceName=default` 且有 `runtimeWorkspace.workspaceId` 的既有个人工作区；找不到 default 私人工作区时返回空态，不调用 `ensureDefaultPersonalWorkspace`、不加载文件树。用户显式点击版本或新增版本后仍会调用 `ensureDefaultPersonalWorkspace(versionId)` 创建或修复 default 私人 worktree，避免首页直接展示或修改应用版本副本。`handleSelectApp → applyManagedWorkspace → switchWorkspace → syncCurrentVersionFromWorkspace` 链路上，`applyManagedWorkspace` 会把 `POST /workspaces/{workspaceId}/recent` 响应中的 `appId` / `versionId` / `applicationWorkspaceId` 通过 `mergeRecentRuntimeResponse` 合并到本次切到的工作区上；`syncCurrentVersionFromWorkspace` 优先用 `workspace.versionId` 直接设回 `currentVersionFromWorkspace`，并按 `workspace.applicationWorkspaceId` 触发对应模板 `versions` 的预加载，使 `WorkbenchFooter.selectedTemplate` 立即找到匹配、左下角"切换工作空间"按钮立刻显示 `模板 / 版本` 名；footer 按钮和版本子菜单同时展示当前私人 worktree 分支，便于确认后续改动落在哪个 `{应用版本分支}_{userId}_default` 分支。`versionId` 缺失时回退到原 `versionsByTemplateId` 精确匹配逻辑以兼容旧数据。`appId` 不在当前账号的应用目录（应用被禁用 / 切换租户）或接口失败时，降级到 `apps[0]`，但不会因此自动加载工作区；空目录则保持空态。
- 左侧文件树上方提供应用工作空间模板、版本和个人空间切换，支持新增版本、创建个人工作区、查看个人/应用差异以及双向同步。
- 工作台左下角"应用工作空间"两级菜单：第一级只显示模板 `workspaceName`（不展示 `directoryPath` / `branch`），hover 模板触发 `GET .../versions` 懒加载子菜单（`directoryPath` / `branch` 也不再展示）。子菜单底部固定一行「+新增版本」，点击后弹 el-dialog（`ElDatePicker` `type=month` / `format=yyyy年M月` / `value-format=yyyy年M月`），确认后调 `POST .../versions` 透传 `yyyy年M月` 字符串，成功后失效 `versionsByTemplateId` 缓存并把新版本切到工作区。
- 工作区文件树加载、打开文件、保存、文件状态、删除和实时预览读取均通过 `@test-agent/backend-api` 的目标后端文件 WebSocket client 完成；切换工作区会关闭旧连接并重新路由到当前用户 opencode 进程同服务器后端。文件 WebSocket 路由和 ticket 只读取 ACTIVE binding、进程归属与后端在线快照，不触发 manager 强健康检查；切换工作区时使用 workspaceId + 加载代次丢弃旧请求结果并取消旧重试定时器，单个目录错误只显示在文件树区域，不覆盖新工作区。发送消息和目录/运行态加载的主判定来自 10 秒弱健康结果；`/processes/me` 只在登录/刷新、头像点击、初始化完成和弱健康不健康时强查并覆盖当前 ready 状态。
- 左侧文件区包含工作区文件树和 `Agent` tab；Agent tab 固定展示“公共级”和“工作空间级”两个根目录，普通工作空间文件树过滤根级 `.opencode` 以避免重复。普通用户只读打开 agent/skill 文件，根级仍可看到 `agents/` 与 `skills/`，但会隐藏 `.DS_Store`、`.gitignore`、`.keep`、`node_modules`、`opencode.jsonc`、`package*.json` 等配置仓库工程杂项；`SUPER_ADMIN` 可更新公共配置、创建 worktree、切换公共 worktree、编辑文件、查看 Agent Git diff、双击文件 stage、提交和发布；公共仓库有未提交修改时仍加载文件树，提交并推送弹窗默认提交当前工作区修改，不再要求先勾选放弃本地修改，只有用户明确勾选“放弃已跟踪本地修改后再提交”时才传 `discardLocalChanges=true`，确认后会像应用工作区提交一样展示拉取远端、暂存提交、合并远端、推送和广播阶段以及当前 Git 命令。工作空间级 `+` 只生成符合当前 opencode skill 模板的应用技能包：`skills/<name>/SKILL.md`、`skills/<name>/rules/README.md`、`skills/<name>/templates/README.md`，`name` 只生成小写字母、数字和短横线，frontmatter 包含 `name`、`description`、`compatibility` 与 `metadata`，不自动生成 Agent。公共级切换先选择已初始化服务器，再选择“直接公共配置目录”或该服务器上的 `ACTIVE` worktree，切换后只清空公共文件树缓存并改变 `worktreeId/linuxServerId` 上下文；公共级根目录加载有请求超时和刷新批次保护，不会因为单次请求卡住而永久显示加载中。Agent 配置文件目录列表、读取和写入均通过 backend-api 的 agent-config 文件 WebSocket route/ticket/RPC，公共直接模式会选择已初始化公共配置服务器并把 `linuxServerId` 随打开的 tab 保存，worktree 模式按落库服务器路由；已打开 tab 不随切换关闭，保存时沿用 tab 内的上下文。创建公共 worktree 弹窗会实时读取远端分支和在线服务器公共配置仓库状态，后端和前端均按稳定 `linuxServerId` 合并在线 Java 快照，避免同一服务器 Java 进程重启窗口内出现重复服务器行；只允许选择已初始化服务器并提交 `linuxServerId`，无已初始化服务器时禁用确认并引导到系统管理初始化。Git 长操作通过 `agent-config` ticket WebSocket 展示进度，不走 RunEvent SSE；个人工作区“提交并推送”会先连接进度 WebSocket，再调用 publish，在弹窗中用 `command` 事件展示当前正在执行的 Git 命令，公共 Agent“提交并推送”会先 fetch 远端，再提交本地修改、merge `origin/{branch}` 并 push；即使本次没有新 commit 也会继续 merge 和 push。远端合并冲突时，公共级 Agents 树刷新只调用轻量冲突文件列表接口来标红冲突文件，不拉取完整 diff patch；公共级 Agents 树和提交失败弹窗都会直接列出冲突文件并提供“处理冲突”、批量保留本地、批量采用远端和取消合并按钮；处理冲突会打开复用 Monaco 的三方冲突编辑器，全部解决后再次提交会完成 merge commit 并推送。Git 远端失败时优先展示后端 `gitFailureHint` 和 `traceId`，worktree 或公共 update-and-push 冲突时展示 `details.conflictFiles` 中的具体冲突文件，不把原始 stderr、完整命令或内部路径直接展示到 UI。
- `SUPER_ADMIN` 用户在文件区底部工作空间切换控件右侧可见服务器切换图标按钮；点击后先选择后端服务器，再通过目标后端 WebSocket 从该 Java 进程运行目录开始浏览目录并添加工作空间。选择器顶部明确展示“将作为工作空间选择”的当前目录，完整展示目录名并换行展示完整路径；下面列表仅表示可进入的子目录，避免误把子目录列表理解为已选目录内容。选中服务器与当前 agent 不同服务器时，目录输入和确认按钮禁用，并提示“工作空间与 agent 不在同一服务器”。
- 点击历史会话时会先用 `session.workspaceContext` 提示的应用/工作区归属调用 `markRecentManagedWorkspace(session.workspaceId)` 校验并回填 `appId/versionId/applicationWorkspaceId`，成功后切换到该会话所属应用和运行态 Workspace；如果用户已不属于历史应用、应用/工作区已不存在或切换失败，则保持当前应用和工作区不变，只恢复 transcript，并把输入框和发送按钮置为只读，悬浮显示具体原因。
