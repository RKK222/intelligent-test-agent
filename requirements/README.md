# 需求与历史归档目录

> ⚠️ **本目录文档仅供历史追溯，不得作为设计或编码依据。**

## 目录性质

本目录存放历史设计文档、需求草案、阶段计划草稿和待办笔记。这些内容：

- **可能已过时**：所描述的技术选型、产品形态、接口规划可能已与实际代码分叉。
- **可能已落地**：其中的设计多数已被代码和 `docs/` 稳定文档取代，重复保留仅为追溯。
- **不再维护同步**：本目录文档不随代码变更更新。

## 编码依据

设计与编码的唯一依据是：

- `AGENTS.md` 和 `docs/README.md` 索引的稳定文档。
- `backend/`、`frontend/` 下的代码现状与各模块/包 README。
- 当代码与本文档冲突时，**以代码和 `docs/` 为准**，本目录任何文档都不构成约束。

## 子目录

- `archive/`：历史设计（`design/`）、阶段计划（`plan/`）、总体方案、前端需求草案、PTY 设计推导等已归档文档。
- `todo/`：当前活跃的待办与后续路线草稿，属过程性笔记，不作为稳定规范。
